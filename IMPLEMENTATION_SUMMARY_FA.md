## 🎯 نحوه کار سیستم

### سناریو 1: تغییر موجودی در سایت 1
```
سایت 1: محصول A = 50 → 40
    ↓
Hook: woocommerce_product_set_stock
    ↓
sync_on_stock_change() → wp_schedule_single_event()
    ↓
3 ثانیه بعد: sync_inventory()
    ↓
get_latest_stock(): 40 (سایت 1) vs 50 (سایت 2)
تاریخ‌ها: سایت 1 تازه‌تر است
    ↓
✅ سایت 2: 40 می‌شود
✅ سایت 1: 40 می‌ماند (قبلاً کم شده بود)
```

### سناریو 2: انتقال محصول
```
سایت 1: محصول جدید (ID 100, SKU "P1")
    ↓
Transfer Manager: transfer_simple_product(100)
    ↓
1. ساده است؟ ✅
2. منتقل نشده؟ ✅
3. ایجاد در سایت 2 → ID 250
4. موجودی = 50 → هر دو سایت = 50
5. Mapping: {site1: 100, site2: 250} ✅
    ↓
✅ اپدیت موجودی هماهنگ‌سازی شود
```

## 🔄 جریان Last-Write-Wins

**تاریخ‌های تغییر کلیدی:**

```php
// get_latest_stock() منطق:

$date1 = strtotime($product1['date_modified']);  // سایت 1
$date2 = strtotime($product2['date_modified']);  // سایت 2

if ($date1 > $date2) {
    return $stock1;  // سایت 1 تازه‌تر است
} else {
    return $stock2;  // سایت 2 تازه‌تر است
}

// اگر هردو یکسان:
return max($stock1, $stock2);  // بیشترین موجودی
```
