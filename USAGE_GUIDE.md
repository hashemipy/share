# راهنمای استفاده - سیستم همگام‌سازی دوطرفه

## صفحات جدید در Admin

### 1. صفحه "وضعیت نقشه‌برداری"
**مسیر:** Inventory Sync → وضعیت نقشه‌برداری

**فرائض:**
- نمایش تمام جفت محصول‌های mapped
- وضعیت هر جفت: ✅ موفق | ❌ خرابی | ⏳ در انتظار
- نمایش قفل: 🔒 قفل شده (در حال sync) | 🔓 باز
- **آخرین تغیر:** کدام سایت تغیر را شروع کرد و چند دقیقه پیش
- **موجودی فعلی:** آخرین موجودی ثبت‌شده
- **دکمه "دوباره تلاش":** برای موارد خرابی

### 2. صفحه "انتقال محصولات"
**مسیر:** Inventory Sync → انتقال محصولات

**فرائض:**
- انتخاب محصولات ساده (فقط ساده، نه متغیر)
- دکمه "انتقال محصولات انتخاب‌شده"
- نمایش وضعیت انتقال

## نحوه کار سیستم

### سناریو 1: تغیر موجودی معمولی
```
1️⃣ موجودی سایت 1 را تغییر دهید (مثلاً 10 → 5)
2️⃣ سیستم خودکار:
   - Cron job فعال می‌شود
   - موجودی سایت 2 = 5 می‌شود
   - ۳۰ ثانیه locked می‌ماند
3️⃣ نتیجه: هر دو سایت موجودی = 5
```

### سناریو 2: تغیر همزمان در دو سایت
```
1️⃣ موجودی سایت 1: 10 → 5
2️⃣ موجودی سایت 2: 10 → 8 (همزمان)
3️⃣ سیستم:
   - سایت 1 sync شود اول (۳۰ ثانیه lock)
   - سایت 2 sync درخواست نادیده گرفته می‌شود
   - بعد از ۳۰ ثانیه: سایت 2 sync می‌شود
4️⃣ نتیجه: آخرین تغیر انجام می‌گیرد
```

### سناریو 3: خرابی و Retry
```
1️⃣ موجودی تغیر: 10 → 5
2️⃣ خطا: سایت 2 دسترسی‌پذیر نیست
3️⃣ سایتم:
   - تلاش دوم بعد از ۱ دقیقه
   - تلاش سوم بعد از ۲ دقیقه
   - تلاش چهارم بعد از ۳ دقیقه
   - اگر ۵+ تلاش ناموفق: متوقف
4️⃣ کاربر:
   - به صفحه "وضعیت نقشه‌برداری" برود
   - وضعیت را ببیند: ❌ خرابی
   - دکمه "دوباره تلاش" را کلیک کند
5️⃣ نتیجه: sync دوباره تلاش می‌شود
```

## بهبودی‌های اصلی

### ✅ حل Ping-Pong Bug
**قبل:** موجودی سایت ۱ تغییر می‌شد، اما بعد برگشت می‌آمد
**بعد:** تغیرات یک‌سمتی هستند - اگر سایت ۱ تغیر کند، سایت ۲ دنبال می‌کند

### ✅ حل Cron تکرار
**قبل:** گاهی sync چند بار اجرا می‌شد
**بعد:** Lock System = فقط ۱ sync برای هر جفت در یک زمان

### ✅ اعلان بهتر
**قبل:** کاربر نمی‌دانست چه شد
**بعد:** صفحه جزئیات + لاگ‌های دقیق

## تنظیم برای بهترین نتیجه

### 1. Cron Job
بهتر است Cron هر ۵ دقیقه اجرا شود:
```
*/5 * * * * wp cron event run --due-now
```

**اگر هر دقیقه اجرا شود:**
- سیستم باز هم کار می‌کند (Lock Protection)
- اما کمتر موثر است

### 2. بررسی منظم
هر روز به "وضعیت نقشه‌برداری" مراجعه کنید:
- موارد خرابی را بررسی کنید
- دکمه "دوباره تلاش" را فشار دهید

### 3. لاگ‌های تفصیلی
اگر مشکل داشتید:
- صفحه "لاگ‌ها" را چک کنید
- نام محصولات، تغییرات، و پیغام‌های خرابی را ببینید

## سوالات متداول

### سوال: چرا موجودی دوباره برگشت؟
**جواب:** معمولاً به دلیل:
1. تغیر همزمان در دو سایت (سیستم فقط یکی را انجام می‌دهد)
2. خرابی API (لاگ‌ها را چک کنید)
3. timeout در cron

**حل:** صفحه "وضعیت نقشه‌برداری" را چک کنید و دوباره تلاش کنید

### سوال: چند بار موجودی sync می‌شود؟
**جواب:** فقط یک بار (Lock System)

اگر cron هر دقیقه اجرا شود:
- دقیقه ۱: sync می‌شود (lock ۳۰ ثانیه)
- دقیقه ۲-۶: skip می‌شود (هنوز lock)
- دقیقه ۷+: فقط نوبت‌های جدید

### سوال: اگر Retry ۵ بار ناموفق بود چی کنم؟
**جواب:**
1. صفحه "لاگ‌ها" را چک کنید برای یافتن مشکل
2. مشکل API را حل کنید
3. محصول را از mapping حذف کن و دوباره اضافه کن

## بهترین تمرین‌ها

1. **تغیرات بزرگ را پایش کنید**
   - هنگام تغیر زیاد موجودی، صفحه "وضعیت نقشه‌برداری" را باز بگذارید
   
2. **Notification تنظیم کنید**
   - email برای موارد خرابی (اگر پلاگین WP Notifications دارید)
   
3. **منظم sync کنید**
   - اگر دستی موجودی تغیر دادید، ۳۰ ثانیه صبر کنید قبل از تغیر دوباره
   
4. **لاگ‌ها را پاک کنید**
   - هر ماه لاگ‌های قدیمی را پاک کنید (برای performance)

## مشکل‌گیری (Troubleshooting)

### مشکل: موجودی sync نمی‌شود
**راه‌حل:**
1. تست اتصال سایت‌ها: صفحه Settings
2. وضعیت Mapping را چک کنید
3. لاگ‌های خرابی را بخوانید
4. API keys را تأیید کنید

### مشکل: Lock برای زمان خیلی طولانی
**راه‌حل:**
1. صفحه "وضعیت نقشه‌برداری" برو
2. Unlock دستی: دوباره تلاش کنید
3. اگر بازهم قفل: database را چک کنید

### مشکل: خطای "تلاش‌های زیاد"
**راه‌حل:**
1. ۵ دقیقه صبر کنید (تالی‌ها پاک می‌شوند)
2. یا دوباره تلاش کنید
3. API را چک کنید
