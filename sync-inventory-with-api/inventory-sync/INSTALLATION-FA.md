# 📖 راهنمای نصب و راه‌اندازی Inventory Sync Pro

## 📋 نیازمندی‌ها

- WordPress 5.0 یا بالاتر
- WooCommerce 4.0 یا بالاتر
- PHP 7.4 یا بالاتر
- دسترسی مدیریتی (Administrator)
- دو سایت WooCommerce جداگانه (یا بیشتر)

## 🎯 مراحل نصب

### مرحله 1: دانلود و نصب پلاگین

#### راه 1: نصب از طریق داشبورد WordPress

1. به **داشبورد WordPress** > **پلاگین‌ها** > **افزودن پلاگین** بروید
2. روی **بارگذاری پلاگین** کلیک کنید
3. فایل `inventory-sync.zip` را انتخاب و آپلود کنید
4. روی **اکنون نصب و فعال کنید** کلیک کنید

#### راه 2: نصب دستی

1. پوشه `inventory-sync` را از فایل‌های پلاگین استخراج کنید
2. از طریق FTP یا File Manager، پوشه را در `/wp-content/plugins/` آپلود کنید
3. به **پلاگین‌ها** بروید و **Inventory Sync Pro** را فعال کنید

### مرحله 2: تایید نصب

✓ پس از فعال‌سازی، باید یک منوی جدید **Inventory Sync** در صفحه‌کنار ظاهر شود

## 🔑 تهیه API Credentials

### برای سایت 1️⃣

1. **ورود به پنل مدیریت** سایت اول
2. **WooCommerce** > **تنظیمات** بروید
3. تب **Advanced** را انتخاب کنید
4. **REST API** را کلیک کنید
5. **Create an API credential** را کلیک کنید

**مراحل تکمیل API:**

| گام | توضیح |
|------|--------|
| Credential Name | نام مناسب وارد کنید (مثل: Inventory Sync) |
| User | Administrator انتخاب کنید |
| Permissions | **Read and Write** انتخاب کنید |
| Description | توصیف اختیاری بنویسید |

6. **Generate API Credentials** را کلیک کنید
7. **Consumer Key** و **Consumer Secret** را **کپی** کنید
8. درجایی ایمن ذخیره کنید (مثل Notepad)

### برای سایت 2️⃣

**همین مراحل را برای سایت دوم تکرار کنید**

## ⚙️ تنظیم در Inventory Sync

### مرحله 1: ورود به تنظیمات

1. **Inventory Sync** > **Dashboard** بروید
2. تب **⚙️ تنظیمات** را کلیک کنید

### مرحله 2: تنظیم سایت 1️⃣

![نام سایت](https://via.placeholder.com/400x50?text=Site+1+Name)

**فیلدهای مورد نیاز:**

```
📍 نام سایت: فروشگاه اصلی
🌐 آدرس سایت: https://site1.com
🔐 API Key: ck_1234567890abcdef
🔐 API Secret: cs_1234567890abcdef
```

**راهنمای پر کردن:**
- **نام سایت**: هر نام دلخواه (برای تشخیص)
- **آدرس سایت**: آدرس کامل WITHOUT slash در انتها
- **API Key/Secret**: از مرحله قبل کپی کنید

### مرحله 3: تنظیم سایت 2️⃣

همین فیلدها را برای سایت دوم پر کنید

### مرحله 4: تست اتصال

**برای سایت 1:**
```
کلیک بر روی [🔗 تست اتصال] 
→ اگر سبز باشد: ✅ اتصال برقرار است
→ اگر قرمز باشد: ❌ مشکل در اتصال
```

**اگر مشکل است:**
1. آدرس سایت را دوبار چک کنید
2. API Key و Secret را صحیح وارد کرده‌اید؟
3. از WordPress Dashboard درحالت Incognito بروید

**برای سایت 2:**
همین فرایند را تکرار کنید

### مرحله 5: تنظیمات پیشرفته

**جهت هماهنگ‌سازی:**
```
سایت 1 → سایت 2  (موجودی از سایت 1 به 2 منتقل می‌شود)
سایت 2 → سایت 1  (موجودی از سایت 2 به 1 منتقل می‌شود)
دوطرفه            (موجودی در دو طرف برابر می‌شود)
```

**هماهنگ‌سازی خودکار:**
```
☐ غیر فعال - فقط دستی
☑ فعال - هر 5 دقیقه خودکار
```

### مرحله 6: ذخیره تنظیمات

کلیک کنید: **💾 ذخیره تنظیمات**

```
✅ پیام موفقیت ظاهر می‌شود
```

## 🔗 مرحله بعدی: مرتبط‌سازی محصولات

### اگر محصولات جدید هستند:

1. به تب **📤 انتقال محصولات** بروید
2. محصولات را انتخاب کنید
3. **📤 انتقال** را کلیک کنید
4. منتظر تکمیل باشید
5. محصولات در سایت 2 اضافه می‌شوند

### اگر محصولات قبلاً موجود هستند:

1. به تب **🔗 مرتبط‌سازی محصولات** بروید
2. محصول سایت 1 و سایت 2 را انتخاب کنید
3. **ذخیره نقشه‌برداری** را کلیک کنید

## 📊 ابتدای استفاده

### روز اول - تست

1. **یک محصول** برای تست انتخاب کنید
2. **موجودی را بروزرسانی** کنید در سایت 1
3. **منتظر 5 دقیقه** باشید (یا اگر خودکار غیرفعال است، **دستی sync** کنید)
4. **سایت 2** را بررسی کنید - موجودی باید تغییر کرده باشد

### چک‌لیست:

```
□ تنظیمات صحیح است
□ اتصال برقرار است
□ محصولات مرتبط شده‌اند
□ یک محصول برای تست هماهنگ کردم
□ موجودی در دو سایت یکسان است
□ لاگ‌ها عملیات موفق را نشان می‌دهند
```

## 🚨 رفع مشکلات رایج

### مشکل: "اتصال ناموفق"

```
❌ درخواست API ناموفق بود

حل:
1. آدرس سایت را بررسی کنید (بدون slash انتها)
2. https:// را استفاده کنید (نه http://)
3. API Key و Secret را دوبار چک کنید
4. در سایت مبدا، API را فعال کردید؟
5. WordPress Cron فعال است؟
```

### مشکل: "محصول منتقل نمی‌شود"

```
❌ خطا در انتقال

حل:
1. تب [📋 لاگ‌ها] را بررسی کنید - پیام خطا چیست؟
2. آیا در سایت 2 اجازه نوشتن دارید؟
3. از یک محصول ساده شروع کنید
4. از WooCommerce Importer استفاده کنید
```

### مشکل: "موجودی sync نمی‌شود"

```
❌ موجودی نگرفت

حل:
1. نقشه‌برداری محصول را بررسی کنید
2. هماهنگ‌سازی خودکار را فعال کنید
3. اگر غیرفعال است، **دستی هماهنگ کنید**
4. لاگ‌ها را بررسی کنید
```
