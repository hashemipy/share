# نصب و استفاده از فیچر جفت‌سازی دو طرفه

## 📦 نصب

### مرحله 1: فعال‌سازی پلاگین

1. وردپرس ادمین خود را باز کنید
2. به **Plugins** بروید
3. **Inventory Sync Pro** را جستجو کنید
4. **Activate** کلیک کنید

### مرحله 2: تنظیمات سایت‌ها

1. منوی **Inventory Sync** را باز کنید
2. به تب **⚙️ تنظیمات** بروید
3. اطلاعات **سایت 1** را وارد کنید:
   - نام سایت
   - آدرس سایت
   - API Key (کلید مصرف کننده WooCommerce)
   - API Secret (سر رمز مصرف کننده WooCommerce)
4. **🔗 تست اتصال** را کلیک کنید
5. تکرار کنید برای **سایت 2**

### مرحله 3: فعال‌سازی Auto Sync

در تب تنظیمات:
- ✅ "Auto Sync" را فعال کنید
- Sync Interval را مطابق نیاز تنظیم کنید (پیش‌فرض: 5 دقیقه)

## 🎯 شروع استفاده

### ایجاد اولین جفت

1. به تب **💑 جفت‌سازی محصولات** بروید
2. **"ایجاد جفت جدید"** را انتخاب کنید
3. محصول‌ها را جستجو کنید و انتخاب کنید
4. **"ایجاد جفت"** را کلیک کنید

### بررسی وضعیت

1. به تب **"مدیریت جفت‌ها"** بروید
2. تمام جفت‌های فعال را ببینید
3. **Sync** کلیک کنید برای sync فوری

### ردیابی لاگ‌ها

1. به تب **"لاگ‌های هماهنگ‌سازی"** بروید
2. تمام عملیات‌های sync را ببینید
3. مشکلات را شناسایی کنید

## ⚙️ تنظیمات پیشرفته

### متغیرهای محیط

```php
// wp-config.php یا .env
define('INVENTORY_SYNC_TIMEOUT', 30); // timeout API به ثانیه
define('INVENTORY_SYNC_RETRY_COUNT', 3); // تعداد تلاش‌های مجدد
define('INVENTORY_SYNC_VERIFY_SSL', true); // تایید SSL
```

### Hooks برای توسعه‌دهندگان

```php
// وقتی جفت ایجاد شود
add_action('inventory_sync_pair_created', function($pair_id) {
    // کد شما
});

// وقتی sync انجام شود
add_action('inventory_sync_pair_synced', function($pair_id) {
    // کد شما
});

// فیلتر برای تغییر موجودی قبل از sync
add_filter('inventory_sync_apply_stock', function($stock, $pair_id) {
    return $stock; // تغییر موجودی
}, 10, 2);
```

## 🔧 عیب‌یابی

### بررسی نمادی

#### ✓ تنظیمات صحیح

```
[✓] API Keys معتبر است
[✓] اتصال سایت‌ها برقرار است
[✓] Database tables ایجاد شده‌اند
[✓] Cron jobs فعال هستند
```

#### ✗ مشکلات معمول

| مشکل | حل |
|------|-----|
| API Connection خطا | API Keys را بررسی کنید |
| Sync کار نمی‌کند | Auto Sync را فعال کنید |
| محصولات پیدا نمی‌شوند | اطمینان حاصل کنید محصول ساده است |
| موجودی sync نمی‌شود | لاگ‌ها را برای خطا بررسی کنید |

```php
// wp-config.php میں اضافه کنید
define('WP_DEBUG', true);
define('WP_DEBUG_LOG', true);
define('WP_DEBUG_DISPLAY', false);

// سپس دیکھیں: /wp-content/debug.log
```

### نمایش سریع

```

📊 کل sync‌ها: 324
⚠️ خطاهای اخیر: 2
📅 آخرین sync: 2 دقیقه پیش
```

### عملیات دستی

- 🔄 **Sync**: برای sync فوری یک جفت
- 🗑️ **حذف**: برای حذف یک جفت
- 📋 **لاگ**: برای دیدن تاریخچه

## 🚀 بهینه‌سازی کارایی

### برای سایت‌های بزرگ

```php
// wp-config.php
define('INVENTORY_SYNC_BATCH_SIZE', 50); // تعداد محصول در هر batch
define('INVENTORY_SYNC_CRON_INTERVAL', 600); // 10 دقیقه
define('INVENTORY_SYNC_CACHE_TIME', 3600); // 1 ساعت
```

### بهترین راهکارها

1. **Cron را صحیح تنظیم کنید**: از real cron استفاده کنید، نه WordPress cron
2. **لاگ‌ها را پاک کنید**: هر ماه لاگ‌های قدیم را حذف کنید
3. **Offload کنید**: محصولات متغیر را از sync خارج کنید
4. **Monitoring**: مراقب خطاها باشید

## 📞 پشتیبانی

### اطلاعات مفید

- **نسخه پلاگین**: 1.1.0
- **نسخه WordPress مورد نیاز**: 5.0+
- **نسخه PHP مورد نیاز**: 7.4+
- **نسخه WooCommerce**: 3.0+

### درخواست کمک

اگر مشکل دارید:

1. لاگ `/wp-content/debug.log` را بررسی کنید
2. API connection را تست کنید
3. قاعده‌های تجدید نویسی را بررسی کنید
4. FTP را برای permissions بررسی کنید
