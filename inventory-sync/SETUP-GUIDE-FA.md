## نحوه کار

### سناریو کامل:

**صبح: سایت 1 شما 100 عدد محصول دارد**
```
سایت 1: محصول A = 100 عدد
سایت 2: محصول A = خالی (هنوز انتقال نیافته است)
```

**گام 1: انتقال محصولات**
- تب "انتقال محصولات" بروید
- محصول A را انتخاب کنید (یا همه را انتخاب کنید)
- روی "انتقال محصولات انتخاب‌شده" کلیک کنید
- سیستم محصول را اسکن می‌کند و به سایت 2 اضافه می‌کند

**نتیجه:**
```
سایت 1: محصول A = 100 عدد
سایت 2: محصول A = 100 عدد (تازه‌ترین)
```

**گام 2: مرتبط‌سازی**
- تب "مرتبط‌سازی محصولات" بروید
- محصول A از سایت 1 را با محصول A از سایت 2 مرتبط کنید
- سیستم یادداشت می‌کند: "این دو یکی هستند"

**نتیجه:**
```
Database Mapping:
site1_product_id: 123
site2_product_id: 456
sync_enabled: 1
```

**گام 3: فروش و هماهنگ‌سازی خودکار**
- شخصی از سایت 1 محصول A را می‌خرد
- موجودی سایت 1 کاهش پیدا می‌کند: 100 → 99
- سیستم **خودکار** این تغییر را می‌بیند
- محصول A در سایت 2 هم: 100 → 99 می‌شود

**نتیجه:**
```
سایت 1: محصول A = 99 عدد
سایت 2: محصول A = 99 عدد (خودکار به‌روز شد)
```

## مراحل نصب

### مرحله 1: نصب پلاگین
1. فولدر `inventory-sync` را کپی کنید
2. در `/wp-content/plugins/` بگذارید
3. وارد داشبورد WordPress شوید
4. Plugins > Installed Plugins
5. "Inventory Sync" را کلیک کنید و فعال کنید

### مرحله 2: دسترسی به پلاگین
1. داشبورد WordPress > WooCommerce (یا Tools)
2. "Inventory Sync" کلیک کنید
3. تب "⚙️ تنظیمات" را ببینید

## تنظیم سایت‌ها

### دریافت API Credentials

#### برای سایت 1:

1. **وارد داشبورد WooCommerce سایت 1 شوید**
2. **WooCommerce > Settings** بروید
3. **Advanced** تب را کلیک کنید
4. **REST API** را کلیک کنید
5. **Create an API Key** کلیک کنید

```
Description: Inventory Sync Connection
Permissions: Read/Write
```

6. بعد از ایجاد:
   - **Consumer Key** را کپی کنید → "API Key" پلاگین
   - **Consumer Secret** را کپی کنید → "API Secret" پلاگین

#### برای سایت 2:
همان مراحل را برای سایت 2 تکرار کنید

### پر کردن فیلدها

**سایت 1 تنظیمات:**
```
نام سایت: فروشگاه اصلی (هر نامی که دوست دارید)
آدرس سایت: https://yoursite1.com
API Key: ck_1234567890abcdef
API Secret: cs_abcdefghij1234567890
```

**سایت 2 تنظیمات:**
```
نام سایت: فروشگاه دوم
آدرس سایت: https://yoursite2.com
API Key: ck_zyxwvutsrq987654321
API Secret: cs_zyxwvutsrq9876543210
```

### تست اتصال

1. سایت 1 تنظیمات را پر کنید
2. روی "🔗 تست اتصال" کلیک کنید
3. پیام "اتصال موفق" ببینید
4. برای سایت 2 تکرار کنید

### تنظیمات پیشرفته

**جهت هماهنگ‌سازی:**

```
سایت 1 ← سایت 2 (توصیه شده)
└─ سایت 1 مرجع است
└─ موجودی سایت 1 مهم‌تر است
└─ سایت 2 همیشه با سایت 1 هم‌خوان است

سایت 2 ← سایت 1
└─ سایت 2 مرجع است
└─ موجودی سایت 2 مهم‌تر است

دوطرفه
└─ موجودی کم‌تر برای هر دو اعمال می‌شود
└─ مثال: سایت 1 = 50، سایت 2 = 30 → هر دو = 30
```

**هماهنگ‌سازی خودکار:**

✓ فعال: هر فروشی خودکار نتیجه به‌روز رسانی اپدیت می‌شود
✗ غیرفعال: فقط دستی می‌توانید sync کنید

## مرتبط‌سازی محصولات

### چرا مرتبط‌سازی مهم است؟

سیستم باید بداند: "محصول SKU:123 از سایت 1" با "محصول SKU:456 از سایت 2" **یکی هستند**

### نحوه مرتبط‌سازی

1. **تب "🔗 مرتبط‌سازی محصولات" بروید**
2. **دو ستون ببینید:**
   - سمت چپ: محصولات سایت 1
   - سمت راست: محصولات سایت 2

3. **محصول سایت 1 را بخش (مثلاً "پیراهن قرمز")**
4. **محصول متطابق سایت 2 را کلیک کنید (مثلاً "پیراهن قرمز سفید")**
5. **سیستم انجام می‌دهد و ذخیره می‌کند**

### مثال واقعی:

```
سایت 1                          سایت 2
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[✓] پیراهن قرمز (SKU:P1)    ←→  [✓] Red T-Shirt (SKU:TS001)
[ ] شلوار سیاه (SKU:P2)     ←→  [ ] Black Pants (SKU:PT001)
[ ] کفش سفید (SKU:P3)       ←→  [ ] White Shoes (SKU:SH001)
```

### ایمپورت/اکسپورت Mapping

اگر مرتبط‌سازی زیادی دارید:
- کل mapping را **اکسپورت** کنید (CSV)
- در Excel تغییر دهید
- **اپلود** کنید دوباره

## انتقال محصولات

### چه چیزی منتقل می‌شود؟

✓ نام محصول  
✓ توضیح کامل  
✓ عکس‌ها  
✓ دسته‌بندی‌ها  
✓ برچسب‌ها  
✓ خصوصیات  
✓ موجودی  

✗ قیمت (باید دستی تنظیم کنید)  
✗ میزان تخفیف  

### مراحل انتقال

1. **تب "📤 انتقال محصولات" بروید**
2. **محصولات را انتخاب کنید:**
   - [ ] محصول 1
   - [ ] محصول 2
   - یا "انتخاب همه" کلیک کنید

3. **روی "📤 انتقال محصولات انتخاب‌شده" کلیک کنید**
4. **صبر کنید - نوار پیشرفت ببینید:**
   ```
   Transferring 45/200 ████░░░░░░ 22%
   Completed: 44
   Failed: 1
   ```

### وضعیت محصولات

**بعد از انتقال:**
- محصول در سایت 2 **"پیش‌نویس"** است
- مدیر باید بررسی کند
- سپس **منتشر** کند

**چرا؟**
- تصاویر ممکن است بهتر کپی شوند
- قیمت‌ها باید دستی تنظیم شوند
- SEO متادیتا بررسی شود

```
┌─────────────────────────────────────┐
│ شخصی محصول را خریدارد (سایت 1)    │
└──────────┬──────────────────────────┘
           │
           ▼
┌─────────────────────────────────────┐
│ سفارش ثبت می‌شود (Order Status: Processing) │
└──────────┬──────────────────────────┘
           │
           ▼
┌─────────────────────────────────────┐
│ پلاگین تشخیص می‌دهد: موجودی تغییر کرد │
└──────────┬──────────────────────────┘
           │
           ▼
┌─────────────────────────────────────┐
│ محصول را از سایت 1 fetch کن         │
│ موجودی: 100 → 99                   │
└──────────┬──────────────────────────┘
           │
           ▼
┌─────────────────────────────────────┐
│ سایت 2 را اپدیت کن                  │
│ موجودی: 100 → 99                   │
└──────────┬──────────────────────────┘
           │
           ▼
┌─────────────────────────────────────┐
│ لاگ موفق: "Sync: Product A"         │
│ "Quantity: 100 → 99"                │
└─────────────────────────────────────┘
```

### جدول Cron

```
نوع Event         |  تکرار
═══════════════════════════════════
Check Stock       |  هر 5 دقیقه
Retry Failed      |  1، 2، 3 دقیقه بعد
Log Cleanup       |  روز یکشنبه ساعت 3 صبح
```

### اگر API باز شود...

```

─────────────────────────

تلاش 2: 1 دقیقه بعد
تلاش 3: 2 دقیقه بعد
Failed: لاگ شود
```

## عیب‌یابی

### خطای "اتصال ناموفق"

**مسئله:** تست اتصال ناموفق است

**حل:**
1. URL صحیح است؟ (https:// شامل است؟)
2. API Key و Secret کپی صحیح است؟
3. درست **تو سایت** فعال است؟

```php
// بررسی کنید
echo "URL: " . get_option('inventory_sync_site1_url');
echo "Key: " . get_option('inventory_sync_site1_key');
```

### موجودی اپدیت نمی‌شود

**مسئله:** هماهنگ‌سازی خودکار کار نمی‌کند

**حل:**
1. آیا "هماهنگ‌سازی خودکار" فعال است?
2. آیا محصولات مرتبط هستند؟
3. آیا WordPress Cron فعال است؟

```php
// بررسی Cron
wp-cli eval 'wp_cron();'
```

### محصول منتقل نشد

**مسئله:** انتقال محصولات ناموفق است

**حل:**
1. تصاویر بیش‌از 5 مگابایت هستند؟
2. API rate limit رسید؟ (صبر کنید 1 ساعت)
3. محصول متنازع دارد؟

## نکات و ترفندها

### 💡 بهترین روش‌ها

1. **ابتدا تست کنید**
   - چند محصول انتقال دهید
   - بررسی کنید سایت 2
   - اگر خوب بود، باقی انتقال دهید

2. **قیمت‌های متفاوت**
   - انتقال = یکی موجودی
   - اما قیمت‌ها می‌توانند متفاوت باشند
   - تنظیم Markup: 20% اضافه کن

3. **Backup بگیرید**
   - قبل از انتقال بزرگ
   - تا اگر مشکلی پیش آمد، بازگردید

### ⚡ بهینه‌سازی

```
API Calls کمتر = سرعت بیشتر
- از "انتقال همه" دستی استفاده کنید
- هماهنگ‌سازی خودکار را برای فروش فعال کنید
- بزرگ‌ترین محصولات را قبل‌تر انتقال دهید
```

## سوالات متداول

**Q: اگر اتصال قطع شود؟**
A: سیستم خودکار دوباره تلاش می‌کند (3 بار)

**Q: آیا موجودی منفی ممکن است؟**
A: خیر، حداقل 0 است

**Q: می‌توانم Sync را لغو کنم؟**
A: بله، تب "تنظیمات" > هماهنگ‌سازی خودکار را غیرفعال کنید

**Q: محصولات حذف شده را می‌سازم؟**
A: خیر، sync فقط موجودی است، نه حذف
