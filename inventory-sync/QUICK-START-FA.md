# شروع سریع - 5 دقیقه

## مراحل اساسی

### 1️⃣ نصب (1 دقیقه)
```
فایل‌ها → /wp-content/plugins/
WordPress Dashboard → Plugins → Activate "Inventory Sync"
```

### 2️⃣ تنظیمات (2 دقیقه)

**سایت 1:**
```
WooCommerce > Settings > Advanced > REST API > Create API Key

سپس:
┌─ Name: Inventory Sync
├─ Consumer Key: ck_xxx
├─ Consumer Secret: cs_xxx
└─ Permissions: Read/Write
```

**در پلاگین:**
```
تب ⚙️ تنظیمات
├─ سایت 1 نام: فروشگاه اصلی
├─ سایت 1 URL: https://site1.com
├─ سایت 1 Key: ck_xxx
└─ سایت 1 Secret: cs_xxx

[🔗 تست اتصال] → ✓ موفق
```

**سایت 2:** (همان مراحل)

### 3️⃣ انتقال محصولات (1 دقیقه)

```
تب 📤 انتقال
├─ ☑️ انتخاب همه
├─ [📤 انتقال همه محصولات]
└─ ⏳ صبر... (نوار پیشرفت)

✓ 50 محصول منتقل شد
```

### 4️⃣ مرتبط‌سازی (1 دقیقه)

```
تب 🔗 مرتبط‌سازی
├─ محصول "پیراهن قرمز" (سایت 1)
└─ کلیک: "Red Shirt" (سایت 2)

✓ مرتبط شدند
```

### 5️⃣ فعال‌سازی خودکار

```
تب ⚙️ تنظیمات > تنظیمات پیشرفته
├─ ☑️ فعال‌سازی هماهنگ‌سازی خودکار
└─ [💾 ذخیره]

✓ از الان موجودی خودکار به‌روز می‌شود
```

```

   └─ موجودی = 99

         └─ سایت 2: محصول A = 99 (خودکار)

✓ تمام شد، دستی کاری نیست!
```

## جهت‌های Sync

### سایت 1 ← سایت 2 (پیشفرض) ✅
```
سایت 1 = مرجع
سایت 2 = دنبال‌کننده

مثال:
Site 1 = 100 → 99
Site 2 = 100 → 99 (همیشه مثل سایت 1)
```

### سایت 2 ← سایت 1
```
سایت 2 = مرجع
سایت 1 = دنبال‌کننده
```

### دوطرفه
```
کم‌تر = برای هر دو

مثال:
Site 1 = 50
Site 2 = 30
نتیجه:
Site 1 = 30 (اینه کم‌تر)
Site 2 = 30
```

## اگر مشکلی پیش آمد:

### ❌ "اتصال ناموفق"
```
☑️ URL صحیح است؟ (https:// شامل)
☑️ API Key/Secret درست کپی شده؟
☑️ محصولات در سایت دیگر وجود دارند؟
```

### ❌ "محصول منتقل نشد"
```
☑️ سایت 2 فضای دیسک دارد؟
☑️ عکس‌ها 5 مگابایت کم‌تر است؟
☑️ API rate limit رسید؟ (صبر کنید)
```

### ❌ "موجودی اپدیت نمی‌شود"
```
☑️ هماهنگ‌سازی خودکار فعال است؟
☑️ محصولات مرتبط هستند؟
☑️ لاگ‌ها را بررسی کنید (تب آخری)
```

---

## لاگ‌ها کجا هستند؟

```
تب 📋 لاگ‌ها
├─ تاریخ
├─ محصول
├─ عملیات
├─ وضعیت (✓ success | ✗ failed)
└─ پیام خطا (اگر ناموفق بود)

مثال:
[2024-06-14 14:32:10]
Product: "پیراهن قرمز"
Operation: "Sync Inventory"
From: "سایت 1" → To: "سایت 2"
Status: ✓ Success
Message: "Quantity: 100 → 99"
```

## قیمت‌ها

⚠️ **قیمت منتقل نمی‌شود!**

```
انتقال = محصول جدید
نام: ✓

موجودی: ✓
```

**راه حل:**
```

```

## بهترین روش‌ها

### ✅ DO
```
▢ ابتدا 5 محصول تست کنید
▢ بررسی کنید سایت 2
▢ اگر خوب بود، باقی انتقال دهید
▢ قیمت‌ها را دستی تنظیم کنید
▢ Backup قبل از عملیات بزرگ
```

### ❌ DON'T
```
✗ همه محصولات یکجا (اگر شبکه ضعیف)
✗ API Key را ذخیره نکنید (امن نیست)
✗ برای 1000 محصول صبر کن بدون Cron
✗ قیمت‌ها را فراموش نکنید
```

## نکات کلیدی

### 1. Mapping = ارتباط بین محصولات
```
محصول SKU:123 (سایت 1) ↔ محصول SKU:456 (سایت 2)
سیستم: "اینها یکی هستند"
```

### 2. Retry Logic = دوباره تلاش
```
اگر API ناموفق:
تلاش 1: فوری
تلاش 2: 1 دقیقه بعد
تلاش 3: 2 دقیقه بعد
```

### 3. Cron = کار‌های برنامه‌ریزی‌شده
```
هر 5 دقیقه: تمام mappings sync شوند
هر فروش: فوری به‌روز رسانی
```
