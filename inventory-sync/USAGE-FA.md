# Inventory Sync - نسخه ۲.۰ بهبود‌شده

یک پلاگین هوشمند برای هماهنگ‌سازی دو سایت WooCommerce به طور خودکار و دستی.

## ویژگی‌های اصلی

### ۱. تشخیص نقش سایت
- تنظیم **سایت ۱** (مبدا - منتقل‌کننده)
- تنظیم **سایت ۲** (مقصد - دریافت‌کننده)
- تنها سایت ۱ دارای دسترسی به مرتبط‌سازی و هماهنگ‌سازی است

### ۲. مرتبط‌سازی محصولات
- رابط کاربری جدید برای انتخاب و اتصال محصولات از دو سایت
- جستجوی فوری برای یافتن محصولات
- نمایش موجودی و SKU محصول

### ۳. هماهنگ‌سازی موجودی
- **بر اساس صفت‌ها (Attributes)**: محصولات متغیر مطابق صفت‌هایشان (مثل سایز، رنگ) به هم متصل می‌شوند
- **محصولات ساده**: موجودی مستقیماً کپی می‌شود
- **محصولات متغیر**: هر variation بر اساس صفت‌های آن‌ها match می‌شود

### ۴. دو روش هماهنگ‌سازی
- **خودکار**: موجودی خودکار هماهنگ می‌شود وقتی تغییر کند
- **دستی**: دکمه "هماهنگ‌سازی" برای کنترل دقیق زمان هماهنگ‌سازی

### ۵. نمایش‌های مختلف
- **تنظیمات**: پیکربندی API برای هر دو سایت
- **مرتبط‌سازی**: اتصال محصولات
- **انتقال**: انتقال محصولات از سایت ۱ به سایت ۲
- **محصولات منتقل‌شده**: لیست محصولاتی که با موفقیت منتقل شدند
- **لاگ‌ها**: تاریخچه تمام عملیات

## راهنمای استفاده

### مرحله ۱: تنظیمات
1. به پنل **WooCommerce > Inventory Sync > تنظیمات** بروید
2. نقش این سایت را انتخاب کنید (سایت ۱ یا سایت ۲)
3. آدرس API و کلید/رمز هر دو سایت را وارد کنید
4. دکمه **تست اتصال** را برای هر دو سایت کلیک کنید

### مرحله ۲: مرتبط‌سازی محصولات
1. به تب **مرتبط‌سازی محصولات** بروید (فقط اگر سایت ۱ هستید)
2. محصول از سایت ۱ را از لیست سمت چپ انتخاب کنید
3. محصول از سایت ۲ را از لیست سمت راست انتخاب کنید
4. دکمه **ایجاد ارتباط** را کلیک کنید
5. برای دیدن تمام مرتبط‌سازی‌ها، به پایین تب رفته و جدول **محصولات مرتبط شده** را ببینید

### مرحله ۳: هماهنگ‌سازی
**خودکار**: اگر در تنظیمات **هماهنگ‌سازی خودکار** را فعال کردید، موجودی خودکار هماهنگ می‌شود

**دستی**: در تب مرتبط‌سازی، دکمه **هماهنگ‌سازی همه مرتبط‌سازی‌ها** را کلیک کنید

## الگوریتم هماهنگ‌سازی برای محصولات متغیر

برای محصولات متغیر (Variable Products) سیستم این الگوریتم را استفاده می‌کند:

```
۱. هر variation از سایت ۱ را بگیر
۲. تمام صفت‌های آن را استخراج کن (مثلاً: سایز=M، رنگ=سیاه)
۳. در سایت ۲ بگرد و variation‌ای بیاب که تمام صفت‌های آن match کنند
۴. موجودی را از سایت ۱ به سایت ۲ کپی کن
```

**مثال**:
- سایت ۱: تی‌شرت، سایز M، رنگ سیاه، موجودی 15
- سایت ۲: تی‌شرت، سایز M، رنگ سیاه، موجودی 5
- نتیجه: موجودی سایت ۲ = ۱۵

## معالجه خطاها و بهبود عملکرد

### خطای ۵۰۲/۵۰۳/۵۰۴
- Plugin خودکار **3 بار** تلاش می‌کند
- بین تلاش‌ها **1، 2، 4 ثانیه** منتظر می‌ماند (Exponential Backoff)
- Timeout **15 ثانیه** است (کمتر از قبل)
- درخواست‌ها **batch** می‌شوند تا تعداد کمتری درخواست ارسال شود

### مشکل دیتابیس
- Plugin **فقط table mappings و logs** را ایجاد می‌کند
- کوئری‌ها optimized هستند و bulk operations استفاده می‌کنند
- درخواست‌های API قابل کنسل‌کردن هستند

## جدول‌های دیتابیس

۱. **wp_inventory_sync_mapping**: ذخیره ارتباطات بین محصولات
۲. **wp_inventory_sync_logs**: لاگ تمام عملیات
۳. **wp_inventory_sync_category_mapping**: ذخیره ارتباطات بین دسته‌بندی‌ها
۴. **wp_inventory_sync_attribute_mapping**: ذخیره ارتباطات بین صفت‌ها
۵. **wp_inventory_sync_products_transferred**: لیست محصولات منتقل‌شده

## API Rate Limiting

اگر سایت شما Rate Limiting دارد:
- Plugin خودکار **429 error** را تشخیص می‌دهد
- بیشتر منتظر می‌ماند (۲، ۴، ۸ ثانیه)
- تلاش را تکرار می‌کند

## نکات مهم

### ایمنی
- تنها کاربران مدیر دسترسی دارند
- API کلید‌ها safe ذخیره می‌شوند
- تمام input‌ها validated می‌شوند

### عملکرد
- جستجو محصولات real-time است
- لاگ‌ها automatic پاک نمی‌شوند (باید دستی پاک کنید)
- Auto-sync هر 5 دقیقه چک می‌شود

### محدودیت‌ها
- تنها SKU matching برای variations بدون صفت‌ها
- محصولات bundle‌ای و آپشن‌دار support نمی‌شوند
- تصاویر محصول خودکار کپی نمی‌شوند

## Troubleshooting

**مشکل**: خطا در تست اتصال
**حل**: 
- API کلید و رمز را چک کنید
- مطمئن شوید که REST API فعال است
- SSL certificate problems را بررسی کنید

**مشکل**: موجودی‌ها خودکار update نمی‌شوند
**حل**:
- مطمئن شوید **هماهنگ‌سازی خودکار** فعال است
- WP-CRON فعال باشد: `define('DISABLE_WP_CRON', false);`
- لاگ‌ها را برای خطا چک کنید

**مشکل**: موجودی غلط synchronize می‌شود
**حل**:
- مطمئن شوید محصولات **manage_stock** فعال دارند
- Attributes محصولات در هر دو سایت یکسان باشند
- Manual sync را دوباره تلاش کنید

## سوالات متداول

**Q: آیا می‌تونم ۲ سایت دیگری اضافه کنم؟**
A: خیر، فعلاً فقط ۲ سایت پشتیبانی شده است

**Q: آیا محصولات جدید خودکار منتقل می‌شوند؟**
A: خیر، باید دستی از تب "انتقال" انتخاب کنید

**Q: چرا برخی variations match نمی‌شوند؟**
A: احتمالاً صفت‌های آن‌ها match نمی‌کنند یا SKU ها متفاوت هستند

**Q: آیا می‌تونم mapping را حذف کنم؟**
A: بله! دکمه "حذف" را در جدول محصولات مرتبط شده کلیک کنید
