خط راهنمایی سیستم مرتبط‌سازی محصولات - Inventory Sync Pro
============================================================

این فایل راهنماهای استفاده از سیستم مرتبط‌سازی را توضیح می‌دهد.

---

## مرحله 1: تعیین نقش سایت

1. وارد داشبورد "Inventory Sync" شوید
2. به تب "⚙️ تنظیمات" بروید
3. بخش "تعیین نقش پلاگین" را پیدا کنید
4. گزینه‌ای انتخاب کنید:
   - "🟢 سایت 1 (مدیریت‌کننده)" - برای سایت اصلی
   - "🔵 سایت 2 (دنبال‌کننده)" - برای سایت دوم
5. دکمه "💾 ذخیره نقش سایت" را فشار دهید

نکته: فقط سایت 1 می‌تواند محصولات را مرتبط کند. سایت 2 فقط تغییرات را دریافت می‌کند.

---

## مرحله 2: پیکربندی تنظیمات سایت‌ها

1. در همان تب "⚙️ تنظیمات" به پایین اسکرول کنید
2. تنظیمات سایت 1:
   - نام سایت (مثال: "فروشگاه اصلی")
   - آدرس سایت (مثال: https://example1.com)
   - API Key
   - API Secret
   - دکمه "🔗 تست اتصال" را فشار دهید برای بررسی

3. تنظیمات سایت 2:
   - نام سایت (مثال: "فروشگاه دوم")
   - آدرس سایت (مثال: https://example2.com)
   - API Key
   - API Secret
   - دکمه "🔗 تست اتصال" را فشار دهید برای بررسی

4. تنظیمات پیشرفته:
   - جهت هماهنگ‌سازی (site1→site2 یا دوطرفه)
   - فعال کردن Auto Sync
   - Interval هماهنگ‌سازی (به ثانیه)

5. دکمه "💾 ذخیره تنظیمات" را فشار دهید

---

## مرحله 3: مرتبط‌سازی محصولات (سایت 1 فقط)

فقط در سایت 1 این کار قابل انجام است:

1. به تب "🔗 مرتبط‌سازی محصولات" بروید
2. محصولات سایت 1 در سمت چپ و سایت 2 در سمت راست نشان داده می‌شوند
3. روند مرتبط‌سازی:
   - 1️⃣ یک محصول از سایت 1 (سمت چپ) را کلیک کنید
   - 2️⃣ یک محصول از سایت 2 (سمت راست) را کلیک کنید
   - 3️⃣ دکمه "🔗 ایجاد مرتبط‌سازی" را فشار دهید

4. تأیید موفقیت:
   - پیام موفقیت نشان داده می‌شود
   - لیست محصولات تازه شده می‌شود

نکته مهم: وقتی دو محصول مرتبط می‌شوند:
- موجودی آن‌ها خودکار هماهنگ می‌شود
- تغییرات در موجودی خودکار منتقل می‌شود

---

## مرحله 4: ویژگی‌های محصولات متغیر (Variable Products)

اگر محصول دارای تغییرات است (مثل سایز، رنگ):

سیستم خودکار:
1. تمام تغییرات محصول را شناسایی می‌کند
2. تغییرات را بر اساس SKU مطابقت می‌دهد
3. موجودی هر تغییر را جداگانه هماهنگ می‌کند

مثال:
- محصول: "کفش"
  - تغییر: سایز 40، رنگ سیاه (SKU: SH40BK)
  - تغییر: سایز 41، رنگ سفید (SKU: SH41WH)

سیستم موجودی هر تغییر را جداگانه sync می‌کند.

---

## مرحله 5: هماهنگ‌سازی موجودی

دو روش وجود دارد:

### روش 1: هماهنگ‌سازی خودکار

اگر "Auto Sync" فعال است:
- هر تغییری در موجودی خودکار انتقل می‌یابد
- زمان انتقال: حدود 3-5 ثانیه

### روش 2: هماهنگ‌سازی دستی

1. به تب "🔗 مرتبط‌سازی محصولات" بروید
2. دکمه "⚡ هماهنگ‌سازی همه موجودی‌ها" را فشار دهید
3. تمام محصولات مرتبط‌شده، موجودی‌شان هماهنگ می‌شود

---

## مرحله 6: انتقال محصولات

این برای انتقال محصولات جدید از سایت 1 به سایت 2 است:

1. به تب "📤 انتقال محصولات" بروید
2. محصولات سایت 1 لیست می‌شوند
3. انتخاب محصولات:
   - محصولات جداگانه را انتخاب کنید
   - یا "انتخاب همه" را فشار دهید
4. دکمه "📤 انتقال محصولات انتخاب شده" یا "📤 انتقال همه محصولات" را فشار دهید
5. پیش‌رفت انتقال نشان داده می‌شود

---

## مرحله 7: بررسی محصولات منتقل‌شده

1. به تب "✅ محصولات منتقل‌شده" بروید
2. تمام محصولات که با موفقیت منتقل شده‌اند، لیست می‌شوند
3. می‌توانید برای محصول جستجو کنید

---

## مرحله 8: بررسی لاگ‌ها

برای رفع‌یابی و بررسی:

1. به تب "📋 لاگ‌ها" بروید
2. تمام عملیات sync، انتقال و خطاها نشان داده می‌شوند
3. برای هر عملیات می‌توانید:
   - تاریخ و زمان
   - نام محصول
   - نوع عملیات
   - سایت مبدا و مقصد
   - وضعیت (موفق/ناموفق)
   - پیام خطا (در صورت وجود)

---

## امنیت و نکات مهم

1. **Prepared Statements**: تمام کوئری‌های دیتابیسی محفوظ‌اند
2. **Input Validation**: تمام ورودی‌ها بررسی می‌شوند
3. **Permission Checks**: فقط کاربران مجاز می‌توانند فعالیت انجام دهند
4. **AJAX Security**: Nonce از WordPress استفاده می‌شود
5. **Error Handling**: تمام خطاها با دقت رفتار می‌شوند

---

## رفع‌یابی مشکلات

### مشکل 1: خطا در "تست اتصال"
- آدرس سایت درست است؟
- API Key و Secret درست است؟
- فایروال یا تنظیمات شبکه مسدود نیست؟
- سایت دوم فعال است؟

### مشکل 2: محصولات بارگذاری نمی‌شوند
- بررسی کنید که محصولات در هر دو سایت وجود دارند
- بررسی لاگ‌ها برای پیام خطا
- دوباره کوش برای بارگذاری

### مشکل 3: Sync کار نمی‌کند
- آیا محصولات مرتبط‌شده‌اند؟
- آیا Auto Sync فعال است؟
- بررسی لاگ‌ها برای خطاهای API
- اتصال اینترنت بررسی شود

### مشکل 4: خطای دیتابیسی
- بررسی فضای دیتابیس
- جداول دیتابیسی ایجاد شده‌اند؟
- بررسی اجازاتی

---

## نکات راهنمایی

1. شروع کنید: تعیین نقش سایت → پیکربندی → تست اتصال
2. محصولات کم: ابتدا با محصولات کم تست کنید
3. منطق SKU: بررسی کنید SKU‌های محصولات متغیر درست است
4. موجودی: قبل از sync، موجودی‌ها را تنظیم کنید
5. لاگ‌ها: برای درک مشکلات، همیشه لاگ‌ها را بررسی کنید

---

برای سؤالات یا مشکلات، لطفاً لاگ‌های دیتابیس و "📋 لاگ‌ها" را بررسی کنید.
