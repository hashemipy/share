# تغییرات انتقال خودکار ویژگی‌ها و دسته‌بندی‌ها

## خلاصه

سیستم Inventory Sync اکنون می‌تواند **به‌صورت خودکار** دسته‌بندی‌ها و ویژگی‌های محصولات را از سایت ۱ به سایت ۲ منتقل کند و علاوه بر این محصولات منتقل‌شده را **علامت‌گذاری** کند.

---

## ویژگی‌های جدید

### ۱. **انتقال خودکار دسته‌بندی‌ها** 🏷️
- دسته‌بندی‌های مادر و فرزند (سلسله‌مراتب) به‌صورت کامل منتقل می‌شود
- اگر دسته‌بندی در سایت ۲ موجود بود، دوباره ایجاد نمی‌شود
- مثال:
  ```
  سایت 1: لباس کودک (مادر) → تیشرت تک (فرزند)
  سایت 2: خودکار ایجاد می‌شود
  ```

### ۲. **انتقال خودکار ویژگی‌ها** 🎨
- ویژگی‌ها (Attributes) مثل رنگ، سایز منتقل می‌شوند
- تمام مقادیر ویژگی (Colors: قرمز، سبز، آبی) منتقل می‌شوند
- اگر ویژگی یا مقدار موجود بود، دوباره ایجاد نمی‌شود
- مثال:
  ```
  سایت 1: 
    - ویژگی: رنگ (قرمز، سبز، آبی)
    - ویژگی: سایز (۳۵، ۴۰، ۴۵، ۵۰)
  
  سایت 2: 
    - خودکار ایجاد می‌شود
  ```

### ۳. **علامت‌گذاری محصولات منتقل‌شده** ✓
- محصولات منتقل‌شده با متا‌داده `_inventory_sync_transferred` علامت‌گذاری می‌شوند
- شامل اطلاعات:
  - شناسه محصول در سایت ۲
  - آدرس سایت ۲
  - تاریخ انتقال
  - شناسه نقشه‌برداری

### ۴. **نمایش بصری وضعیت انتقال** 👁️
- تب "انتقال محصولات" اکنون ستونی برای "وضعیت انتقال" دارد
- محصولات منتقل‌شده نمایش داده می‌شوند: ✓ منتقل شده (سبز)
- محصولات منتقل‌شده غیرفعال می‌شوند (نمی‌توانند دوباره انتقال پیدا کنند)

## فایل‌های تغییر‌یافته

### 1️⃣ `includes/class-api.php` 
**متد‌های جدید برای دسته‌بندی‌ها و ویژگی‌ها:**
- `get_categories()` - دریافت تمام دسته‌بندی‌ها
- `get_category_by_name()` - جستجو بر اساس نام
- `create_category()` - ایجاد دسته‌بندی جدید
- `get_attributes()` - دریافت تمام ویژگی‌ها
- `get_attribute_by_name()` - جستجو بر اساس نام
- `create_attribute()` - ایجاد ویژگی جدید
- `get_attribute_terms()` - دریافت مقادیر ویژگی
- `get_attribute_term_by_name()` - جستجو بر اساس نام
- `create_attribute_term()` - ایجاد مقدار ویژگی
- `update_product_meta()` - به‌روز‌رسانی متا‌داده محصول

### 2️⃣ `includes/class-sync-manager.php`
**بهبود متد `transfer_product()`:**
- ۱. انتقال دسته‌بندی‌ها (مرحله ۱)
- ۲. انتقال ویژگی‌ها (مرحله ۲)
- ۳. ایجاد محصول (مرحله ۳)
- ۴. انتقال متغیّرها (مرحله ۴)
- ۵. ذخیره نقشه‌برداری (مرحله ۵)
- ۶. علامت‌گذاری محصول (مرحله ۶)

**متد‌های کمکی جدید:**
- `sync_categories()` - انتقال دسته‌بندی‌های سلسله‌مراتبی
- `sync_attributes()` - انتقال ویژگی‌ها و مقادیرشان

### 3️⃣ `includes/class-database.php`
**افزودن ستون‌های جدید:**
- `transferred` - آیا محصول منتقل شده است
- `transfer_verified` - آیا انتقال تأیید شده است

### 4️⃣ `includes/class-category-helper.php` (NEW)
**کلاس جدید برای مدیریت دسته‌بندی‌های سلسله‌مراتبی:**
- `get_category_hierarchy()` - دریافت سلسله‌مراتب کامل
- `sync_category_with_hierarchy()` - انتقال دسته‌بندی و والدینش

### 5️⃣ `admin/dashboard.php`
**افزودن ستون جدید:**
- ستون "وضعیت انتقال" در جدول انتقال محصولات

### 6️⃣ `assets/js/admin.js`
**بهینه‌سازی `renderTransferTable()`:**
- نمایش وضعیت انتقال (✓ منتقل شده)
- غیرفعال کردن محصولات منتقل‌شده

### 7️⃣ `assets/css/admin.css`
**افزودن استایل‌های جدید:**
- `.transferred-badge` - استایل برای نشانگر منتقل‌شده
- `.transfer-products tr:has(input:disabled)` - رنگ‌های ویژه برای ردیف‌های غیرفعال

## جریان انتقال (Flow)

```
محصول از سایت 1 انتخاب می‌شود
           ↓
1️⃣ دسته‌بندی‌ها منتقل می‌شوند (مادر ← فرزند)
           ↓
2️⃣ ویژگی‌ها و مقادیرشان منتقل می‌شوند
           ↓
3️⃣ محصول والد ایجاد می‌شود
           ↓
4️⃣ متغیّرهای محصول ایجاد می‌شوند
           ↓
5️⃣ نقشه‌برداری ذخیره می‌شود
           ↓
6️⃣ محصول اصلی علامت‌گذاری می‌شود
           ↓
✓ انتقال کامل شد!
```

## نمونه‌های استفاده

### مثال ۱: انتقال محصول ساده
```php
$sync_manager = Inventory_Sync_Manager::get_instance();
$result = $sync_manager->transfer_product(123); // 123 = شناسه محصول سایت 1

// اگر موفق بود، محصول جدید در سایت 2 ایجاد شده و دسته‌بندی‌های آن نیز منتقل شده‌اند
```

### مثال ۲: انتقال محصول متغیّر
```php
// محصول متغیّری با ویژگی‌های رنگ و سایز
// دسته‌بندی: لباس → تیشرت
$result = $sync_manager->transfer_product(456);

// نتیجه:
// 1. دسته‌بندی "لباس" ایجاد شد (اگر نبود)
// 2. زیردسته "تیشرت" ایجاد شد
// 3. ویژگی "رنگ" با مقادیر ایجاد شد
// 4. ویژگی "سایز" با مقادیر ایجاد شد
// 5. محصول متغیّر ایجاد شد
// 6. تمام متغیّرها ایجاد شدند
// 7. محصول علامت‌گذاری شد
```

## نکات مهم

⚠️ **نیازها:**
- WooCommerce REST API باید فعال باشد
- کلیدهای API باید صحیح تنظیم شده باشند
- سایت ۲ باید دارای دسترسی نوشتن برای محصولات باشد

🎯 **بهترین شیوه:**
- قبل از انتقال محصولات جدید، تنظیمات API را تست کنید
- محصولات را در دسته‌های کوچک منتقل کنید
- لاگ‌ها را برای خرابی‌های احتمالی بررسی کنید

💡 **نکات فنی:**
- Category Helper از استراتژی caching استفاده نمی‌کند (می‌توان بهبود داد)
- تمام درخواست‌های API retry 3 بار می‌شوند
- متا‌داده انتقال به صورت JSON ذخیره می‌شود

---

## آزمایش

برای تست کردن این ویژگی‌های جدید:

1. ابتدا **تنظیمات API** را تأیید کنید
2. **۱ محصول ساده** منتقل کنید و بررسی کنید:
   - آیا محصول در سایت ۲ ایجاد شد؟
   - آیا دسته‌بندی موجود است؟
3. **۱ محصول متغیّر** منتقل کنید:
   - آیا ویژگی‌ها ایجاد شدند؟
   - آیا متغیّرها صحیح ایجاد شدند؟
4. **دوباره انتقال** را امتحان کنید:
   - آیا محصول غیرفعال است؟
