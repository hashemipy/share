# راهنمای انتقال متغیّرها

## فرایند انتقال متغیّر

```
متغیّر در سایت 1
│
├─ SKU
├─ قیمت عادی (regular_price)
├─ قیمت تخفیفی (sale_price)
├─ موجودی (stock_quantity)
├─ وضعیت موجودی (stock_status)
├─ ویژگی‌های متغیّر (attributes)
│  └─ رنگ: سبز
│  └─ سایز: بزرگ
└─ تصویر (image)
   │
   ↓ انتقال
   │
متغیّر در سایت 2 ✓

```

## داده‌های هر متغیّر

### الزامی (Required)
| فیلد | مثال | توضیح |
|------|------|-------|
| `sku` | `PROD-001-GREEN-L` | شناسه منحصر‌به‌فردی محصول |
| `attributes` | `[{id: 1, option: 'سبز'}, {id: 2, option: 'بزرگ'}]` | ویژگی‌ها و مقادیر آن‌ها |
| `regular_price` | `150000` | قیمت اصلی |

### اختیاری (Optional)
| فیلد | مثال | توضیح |
|------|------|-------|
| `sale_price` | `120000` | قیمت با تخفیف |
| `stock_quantity` | `50` | تعداد موجودی |
| `manage_stock` | `true` | آیا موجودی مدیریت شود |
| `image` | `{src: 'https://...'}` | تصویر متغیّر |
| `description` | `"متغیّر سبز بزرگ"` | توضیح متغیّر |

## مثال عملی

## فرایند تفصیلی

### مرحله 1: دریافت متغیّرها
```php
$variations = $this->site1_api->get_product_variations($product_id, 100, $page)
```
- دریافت تمام متغیّرها با صفحه‌بندی (100 تا 100)

### مرحله 2: آماده‌سازی داده‌های هر متغیّر
```php
$prepared = $this->prepare_variation_data($variation)
```
شامل:
- **قیمت**: regular_price, sale_price
- **موجودی**: stock_quantity, stock_status, manage_stock
- **ویژگی‌ها**: attributes (با ID صحیح)
- **تصویر**: image.src

### مرحله 3: آماده‌سازی ویژگی‌های متغیّر
```php
$attrs = $this->prepare_variation_attributes($variation['attributes'])
```
تبدیل به:
```
[
  {'id': 1, 'option': 'سبز'},
  {'id': 2, 'option': 'کوچک'}
]
```

### مرحله 4: ایجاد گروهی (Batch)
```php
$this->site2_api->batch_create_variations($parent_id, $variations)
```
- ایجاد تمام متغیّرها در یک درخواست API

## مشکلات رایج

### مشکل 1: ویژگی‌ها مطابقت نداشت
**علامت**: متغیّر ایجاد نمی‌شود
**علت**: ویژگی سایت 2 با سایت 1 متفاوت است
**حل**: 
```
1. ویژگی‌های سایت 2 را بررسی کنید
2. SKU را بروزرسانی کنید
3. دوباره انتقال کنید
```

### مشکل 2: موجودی نقل نمی‌شود
**علامت**: متغیّر ایجاد می‌شود اما stock_quantity = 0
**علت**: manage_stock غیرفعال است
**حل**:
```
1. sایت 2 → تنظیمات محصول → مدیریت موجودی را فعال کنید
2. دوباره انتقال کنید
```

### مشکل 3: تصویر منتقل نمی‌شود
**علامت**: متغیّر بدون تصویر ایجاد می‌شود
**علت**: URL تصویر سایت 1 در سایت 2 قابل دسترسی نیست
**حل**:
- WooCommerce خودکار تصاویر را دانلود و ذخیره می‌کند
- اگر خطا داد، URL را دستی اصلاح کنید

## لاگ‌های مهم

هر مرحله لاگ می‌شود:

| لاگ | معنی |
|------|------|
| `get_variations_error` | خطا در دریافت متغیّرها از سایت 1 |
| `no_variations` | هیچ متغیّری برای انتقال وجود ندارد |
| `create_variations_start` | شروع ایجاد متغیّرها |
| `batch_create_variations_error` | خطا در ایجاد متغیّرها |
| `variations_created` | متغیّرها با موفقیت ایجاد شدند |

## نکات مهم

1. **ترتیب اهمیت**:
   - ویژگی‌ها (باید موجود باشند!)
   - قیمت
   - موجودی
   - تصویر

2. **موجودی**:
   - اگر `manage_stock = false`، `stock_quantity` نفرستاده می‌شود
   - محصول در صورت خالی بودن، `instock` باقی می‌ماند

3. **SKU**:
   - باید منحصر‌به‌فردی باشد
   - نمی‌تواند تکراری باشد

4. **تصویر**:
   - URL باید از سایت 1 قابل دسترسی باشد
   - WooCommerce خودکار دانلود می‌کند

## تست کردن انتقال

### 1. یک محصول متغیّر انتخاب کنید
```
سایت 1 → محصول متغیّر با 3-5 متغیّر
```

### 2. انتقال دهید
```
صفحه انتقال → انتخاب محصول → "انتقال"
```

### 3. بررسی سایت 2
```
سایت 2 → محصول جدید (Draft)
│
├─ ویژگی‌ها صحیح؟ ✓
├─ متغیّرها ایجاد شدند؟ ✓
├─ قیمت درست؟ ✓
├─ موجودی درست؟ ✓
└─ تصویر موجود؟ ✓
```

### 4. لاگ‌ها بررسی کنید
```
صفحه لاگ → جستجوی نام محصول
│
└─ variations_created: ✓ موفق
```
