# 🔧 اصلاحات اعمال‌شده - Inventory Sync Plugin

## مشکل 1️⃣: متغیرهای محصول منتقل نمی‌شوند ❌ → ✅

### مسئله:
- هنگام انتقال محصول متغیر از سایت ۱ به سایت ۲، تمام اطلاعات (دسته‌بندی، ویژگی‌ها، عکس‌ها) منتقل می‌شدند
- **اما متغیرهای محصول (Variations) منتقل نمی‌شدند!**
- هر متغیر شامل: قیمت، موجودی، تعداد موجودی، عکس، و ویژگی‌های خاص است

### علت ریشه‌ای:
- ویژگی‌های متغیرها با ویژگی‌های محصول والد Mapping نمی‌شدند
- هنگام انتقال متغیر، سیستم نمی‌دانست کدام ویژگی سایت ۲ را باید استفاده کند

### حل:
```
1. ایجاد mapping واسطی برای ویژگی‌ها:
   - قبل از انتقال متغیرها، تمام ویژگی‌های محصول والد (سایت ۱) 
   - را پیدا می‌کنیم و آن‌ها را به ویژگی‌های سایت ۲ map می‌کنیم
   
2. انتقال داده‌های صحیح:
   - از این mapping هنگام آماده‌سازی داده‌های متغیر استفاده می‌کنیم
   - اطمینان می‌دهیم که هر متغیر با ویژگی‌های صحیح ایجاد شود
```

### تغییرات فایل:
- **class-sync-manager.php**:
  - اضافه شد: `attribute_mapping` در `transfer_variations()`
  - تغییر: `prepare_variation_data()` - حالا `$attribute_mapping` را قبول می‌کند
  - تغییر: `prepare_variation_attributes()` - از mapping استفاده می‌کند

---

## مشکل 2️⃣: هماهنگ‌سازی موجودی متغیرها ❌ → ✅

### مسئله:
- اگر موجودی یک متغیر در سایت ۱ تغییر کند، موجودی در سایت ۲ هماهنگ نمی‌شود
- مشکل: سیستم **نمی‌دانست کدام متغیر در سایت ۲ مطابق کدام متغیر در سایت ۱ است**

### علت ریشه‌ای:
- **هیچ Mapping جدول برای متغیرها وجود نداشت!**
- تنها SKU برای matching استفاده می‌شد (ناقص و نامطمئن)

### حل:
```
1. ایجاد جدول: warehouse_inventory_sync_variation_mapping
   - site1_variation_id ← site2_variation_id
   - SKU و دیگر اطلاعات برای ردیابی
   
2. هنگام انتقال محصول:
   - تمام mapping‌های متغیر ذخیره می‌شود
   
3. هنگام هماهنگ‌سازی موجودی:
   - ابتدا جدول mapping را جستجو می‌کنیم (دقیق)
   - سپس SKU را (fallback/پشتیبان)
   - و درنهایت جدول را به‌روز می‌کنیم
```

### تغییرات فایل:
- **class-database.php**:
  - اضافه شد: جدول `inventory_sync_variation_mapping`
  - اضافه شد: متدهای:
    - `add_variation_mapping()` - ذخیره mapping
    - `get_variation_mapping()` - جستجو بر اساس site1_id
    - `get_variation_mapping_by_sku()` - جستجو بر اساس SKU
    - `update_variation_sync_status()` - به‌روز‌رسانی وضعیت

- **class-sync-manager.php**:
  - تغییر: `sync_variable_stock()` - حالا از جدول mapping استفاده می‌کند
  - اضافه شد: `save_variation_mappings()` - ذخیره mapping‌ها بعد از انتقال
  - تغییر: `save_mapping()` - بازگرداندن mapping_id برای استفاده در variation

## مشکل 3️⃣: مرتبط‌سازی ضعیف و نامشخص ❌ → ✅

### مسئله:
- تب "مرتبط‌سازی محصولات" بسیار نامشخص بود
- کاربر نمی‌دانست محصولات چطور مرتبط می‌شوند
- راه‌ای برای مشاهده mapping‌های موجود نبود

### حل:
```
1. UI بهبود یافت:
   - نمایش جدول تمام mapping‌های فعال
   - نمایش ID، SKU، وضعیت، و تاریخ آخرین هماهنگ‌سازی
   - اضافه شد: دکمه "هماهنگ‌سازی دستی" برای هر mapping
   
2. توضیحات بهتر:
   - شرح‌وار: چطور محصولات مرتبط می‌شوند
   - اشاره‌ای به جداول database
   - راهنمایی برای کاربران
```

### تغییرات فایل:
- **dashboard.php**:
  - تغییر: تب mapping - با نمایش جدول mapping‌ها
  - اضافه شد: جعبه اطلاعات (info box) برای توضیح
  
- **admin.js**:
  - اضافه شد: `loadMappings()` - بارگذاری mapping‌ها
  - اضافه شد: `renderMappings()` - نمایش جدول mapping‌ها
  - تغییر: `syncAllInventory()` - اتصال صحیح به server

## نکات مهم برای آینده:

✅ **هماهنگ‌سازی متغیرها** - حالا کاملاً فعال است
✅ **ردیابی دقیق** - از جدول mapping برای هر متغیر استفاده می‌کند
✅ **Fallback Safety** - اگر mapping پیدا نشود، از SKU استفاده می‌کند

## آزمایش موصیف:

1. **انتقال محصول متغیر**:
   - محصولی را از سایت ۱ به سایت ۲ منتقل کنید
   - ✅ متغیرها باید منتقل شوند
   - جدول `wp_inventory_sync_variation_mapping` را بررسی کنید

2. **تغییر موجودی**:
   - موجودی یک متغیر در سایت ۱ را تغییر دهید
   - ✅ موجودی سایت ۲ باید به‌روز شود
   - لاگ‌ها را در تب "لاگ‌ها" بررسی کنید

3. **مرتبط‌سازی**:
   - به تب "مرتبط‌سازی محصولات" بروید
   - ✅ تمام mapping‌های فعال نمایش داده شود
