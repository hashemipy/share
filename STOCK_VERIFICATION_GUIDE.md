# راهنمای تشخیص و تایید موجودی درست

## مشکل اصلی
گاهی موجودی‌ها غلط اعمال می‌شدند، زیرا سیستم نمی‌دانست کدام موجودی **آخرین** است.

## حل جدید: Timestamp مقایسه دقیق + Verification Logs

### 1. چگونه سیستم تشخیص می‌دهد کدام موجودی آخرین است؟

```
if تاریخ‌ها موجود نیستند → بیشترین موجودی انتخاب شود
if تفاوت < 1 ثانیه → احتمالاً یکسان اند، بیشترین انتخاب
if تفاوت < 2 دقیقه → نزدیک اند، بیشترین انتخاب
if تفاوت > 2 دقیقه → آخرین تغیر انتخاب
```

### 2. صفحه "بررسی دقیق Logs" نحوه کار:

وقتی sync صورت می‌گیرد، سیستم log می‌کند:
- کدام موجودی انتخاب شد؟
- چرا این موجودی انتخاب شد؟
- موجودی قبل چند بود؟
- موجودی بعد چند شد؟

#### مثال Log:

```
ID محصول: 123
نام: محصول A
سایت: سایت 1 → سایت 2
موجودی قبل: 50
موجودی بعد: 100
دلیل انتخاب: سایت 1 آخرین‌تر است (تفاوت: 5 دقیقه)
زمان: 2025-06-18 14:32:45
وضعیت: ✓ موفق
```

### 3. چگونه تشکیل را بررسی کنی:

1. به Admin برو
2. **Inventory Sync → بررسی دقیق Logs** را کلیک کن
3. جدول را بررسی کن:
   - اگر **دلیل انتخاب** منطقی است = ✓ درست
   - اگر **موجودی بعد** صحیح نیست = ❌ مشکل

### 4. علل احتمالی مشکلات:

| علت | دلیل | حل |
|-----|-----|-----|
| تاریخ تغیر غلط | API تاریخ را غلط ارسال می‌کند | بررسی API WooCommerce |
| تفاوت زمانی زیاد | سرورها هم‌وقت نیستند | تنظیم Cron هماهنگ‌سازی |
| موجودی نادرست | Database خراب | اعمال مجدد از Admin |

### 5. فایل Error Logs:

تمام اطلاعات تشخیص‌دهی در این فایل:
```
/wp-content/debug.log
```

خط‌های که شامل `[Inventory-Sync]` باشند:

```
[18-Jun-2025 14:32:45 UTC] [Inventory-Sync] موجودی انتخابی (ID1:123, ID2:456): {"stock":100,"source":"سایت 1","reason":"سایت 1 آخرین‌تر است"}
[18-Jun-2025 14:32:45 UTC] [Inventory-Sync] جزئیات: سایت 1 موجودی=100 (تاریخ:2025-06-18 14:32:00), سایت 2 موجودی=50 (تاریخ:2025-06-18 14:27:00)
```

### 6. نکات مهم:

✓ System اکنون **۱۰۰٪ ثبت می‌کند** که چه انتخابی کرده
✓ می‌توانی به راحتی **تشکیل خودت بررسی کنی**
✓ اگر خطا بود، **دلیل مشخص است**
✓ موجودی **ضرورتاً درست** اعمال می‌شود (زمان‌های معلوم)

## خلاصه:

**قبل:** مشکل - موجودی برمی‌گشت یا غلط اعمال می‌شد

**اکنون:** ✓ نقطه‌ای - هر sync با دلیل ثبت می‌شود و می‌توانی تشکیل بررسی کنی
