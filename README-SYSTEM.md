# سیستم هماهنگ‌سازی انبار - دو پلاگین

## معماری سیستم

### سایت 1 - Master Plugin
**نقش:** مدیریت و کنترل محصولات

مشخصات:
- تب "مرتبط‌سازی محصولات" برای انتخاب و ارسال محصولات
- دریافت Webhook از سایت 2 برای تحدیث موجودی
- Dashboard با آمار Sync
- API Endpoints برای ارتباط

### سایت 2 - Slave Plugin
**نقش:** دریافت و فروش محصولات

مشخصات:
- تب "محصولات دریافت‌شده" - محصولات import شده از سایت 1
- درصد‌گذاری قیمت - اضافه‌کردن درصد به همه قیمت‌ها
- Webhook Sender - ارسال تغییرات موجودی به سایت 1
- خودکار Import - دریافت محصولات ساده و متغیر

---

## نصب و تنظیم

### 1️⃣ بارگذاری پلاگین‌ها

**سایت 1 (Master):**
```
۱. فایل‌های پلاگین را در /wp-content/plugins/inventory-sync-master/ کپی کنید
۲. داشبورد وردپرس → پلاگین‌ها → Inventory Sync Master را فعال کنید
```

**سایت 2 (Slave):**
```
۱. فایل‌های پلاگین را در /wp-content/plugins/inventory-sync-slave/ کپی کنید
۲. داشبورد وردپرس → پلاگین‌ها → Inventory Sync Slave را فعال کنید
```

### 2️⃣ دریافت API Credentials

**در سایت 2:**
- به "محصولات Slave" → "تنظیمات" بروید
- **API Key** و **API Secret** و **Webhook URL** را کپی کنید

### 3️⃣ تنظیم سایت 1

**در سایت 1:**
- به "هماهنگ‌سازی انبار" → "تنظیمات" بروید
- **آدرس سایت 2**: https://site2.com
- **API Key**: (از سایت 2 کپی شده)
- **API Secret**: (از سایت 2 کپی شده)
- دکمه "ذخیره تنظیمات" را کلیک کنید

## نحوه استفاده

### ارسال محصول از سایت 1 به سایت 2

**در سایت 1:**
1. "هماهنگ‌سازی انبار" → "مرتبط‌سازی محصولات"
2. محصول ساده یا متغیر را انتخاب کنید
3. دکمه "ارسال" را کلیک کنید
4. محصول به سایت 2 منتقل می‌شود

### تغییر قیمت در سایت 2

**در سایت 2:**
1. "محصولات Slave" → تب اصلی
2. "درصد افزایش قیمت" را تنظیم کنید
   - مثال: 10 = ۱۰ درصد افزایش
   - مثال: -5 = ۵ درصد کاهش
3. "ذخیره" کلیک کنید
4. تمام قیمت‌های محصولات به‌روز می‌شوند

### نظارت موجودی

**سایت 1 → Dashboard:**
- نمایش آخرین Sync‌ها
- تعداد کل Mapping‌ها
- تعداد Mapping‌های فعال

**سایت 2 → Dashboard:**
- نمایش محصولات دریافت‌شده
- موجودی فعلی هر محصول
- قیمت‌های تطبیق‌شده

## جزئیات انتقال محصولات

### محصولات ساده
✅ نام و توضیح
✅ قیمت معمولی و فروش
✅ SKU
✅ موجودی
✅ تصویر اصلی
✅ وضعیت: همیشه "منتشر شده"

### محصولات متغیر
✅ نام و توضیح
✅ تمام Variations
✅ Attributes برای هر variation
✅ قیمت‌های جداگانه
✅ موجودی‌های جداگانه
✅ تصاویر Variations
✅ وضعیت: همیشه "منتشر شده"

### تطبیق قیمت
```
قیمت‌شده = قیمت اصلی × (1 + درصد / 100)

مثال:
- قیمت اصلی: ۱۰۰۰ تومان
- درصد: ۲۰%
- قیمت‌شده: ۱۰۰۰ × (1 + 20/100) = ۱۲۰۰ تومان
```

## Webhook - بروزرسانی موجودی

### هنگام فروش در سایت 2

جریان:
1. محصول فروخته می‌شود
2. سایت 2 تغییر موجودی را تشخیص می‌دهد
3. Webhook ارسال می‌شود به سایت 1
4. سایت 1 موجودی محصول اصلی را بروز می‌رسانی می‌کند

### URL Webhook (در Master)
```
POST /wp-json/inventory-sync-master/v1/webhook/stock
Headers:
  X-API-Key: (API Key از Slave)
  X-API-Secret: (API Secret از Slave)
Body:
{
  "remote_id": "محصول ID",
  "quantity": 10
}
```

## استکفایی‌سازی (Troubleshooting)

### مشکل: محصولات نمایش داده نمی‌شوند

**راه‌حل:**
1. تنظیمات سایت 1 را بررسی کنید
2. API Key و Secret صحیح هستند؟
3. آدرس سایت 2 صحیح است؟
4. فایروال سایت 2 درخواست‌ها را مسدود نمی‌کند؟

### مشکل: قیمت‌ها به‌روز نمی‌شوند

**راه‌حل:**
1. Markup مثبت یا منفی باشد؟
2. محصول قبلاً import شده است؟
3. کش وردپرس را پاک کنید

### مشکل: موجودی تحدیث نمی‌شود

**راه‌حل:**
1. API credentials صحیح هستند؟
2. سایت 2 به سایت 1 دسترسی دارد؟
3. Logs را بررسی کنید: wp-content/debug.log
