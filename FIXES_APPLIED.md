# اصلاحات اعمال‌شده برای مشکلات انتقال

## خطاهای حل‌شده

### 1. خطای SKU تکراری ✓
**مشکل:** محصول منتقل نمی‌شود چون SKU قبلاً موجود است
```
{"code":"woocommerce_rest_product_not_created",
 "message":"The product with SKU (115) you are trying to insert is already present in the lookup table"}
```

**راه‌حل:**
- متد `product_exists_by_sku()` اضافه شد برای بررسی SKU قبلی
- اگر محصول موجود است، **بروز‌رسانی** می‌شود (نه ایجاد جدید)
- اگر SKU خالی است، یک SKU منحصر‌به‌فردی تولید می‌شود

### 2. دسته‌بندی‌ها منتقل می‌شوند اما محصول نه ✓
**مشکل:** `sync_product_categories()` کار می‌کند اما `transfer_product()` خراب است

**علت اصلی:**
```php
// غلط ❌
'categories' => $this->prepare_terms($product1['categories'] ?? []),

// درست ✓
'categories' => $this->prepare_terms_as_ids($product1['categories'] ?? []),
```

WooCommerce API نیاز به **ID دسته‌بندی** دارد، نه نام!

**راه‌حل:**
- متد جدید `prepare_terms_as_ids()` اضافه شد
- دسته‌بندی‌ها را به صورت `[{'id': 10}]` فرستاده می‌شود

### 3. ویژگی‌های صحیح نشناخته نمی‌شوند ✓
**مشکل:** ویژگی‌های محصول اگر در سایت 2 موجود نیستند، ایجاد نمی‌شوند

**راه‌حل:**
- کلاس `Inventory_Sync_Category_Attribute_Sync` مسئولیت دارد
- قبل از انتقال محصول، تمام ویژگی‌ها ایجاد می‌شوند
- تمام **تکیه‌های ویژگی** (colors, sizes) منتقل می‌شوند

## تغییرات اصلی

### فایل‌های اصلاح‌شده

#### `class-sync-manager.php`
- بهبود متد `transfer_product()` - 45 خط تغییر
- اضافه کردن لاگ‌های دیتیلی
- افزودن SKU validation
- متد `prepare_terms_as_ids()` جدید

#### `class-api.php`
- متد `product_exists_by_sku()` - بررسی SKU
- متد `find_product_by_name()` - جستجوی محصول
- متد `update_product()` - بروز‌رسانی محصول

#### `class-category-attribute-sync.php`
- بهبود `sync_product_categories()` - caching mapping
- افزودن بررسی mapping قبلی

## فرایند انتقال بروز‌شده

```
محصول انتخاب شود
         ↓
[1] SKU بررسی → موجود؟ نه ✓
         ↓
[2] دسته‌بندی‌ها منتقل
         ↓
[3] ویژگی‌ها منتقل (رنگ، سایز، ...)
         ↓
[4] تکیه‌های ویژگی ایجاد (سبز، بنفش، لارج، ...)
         ↓
[5] اطلاعات محصول آماده (name, price, images)
         ↓
[6] دسته‌بندی‌های نقشه‌برداری شده به محصول اضافه
         ↓
[7] محصول ایجاد ✓ (یا بروز‌رسانی اگر موجود بود)
         ↓
[8] متغیّرها ایجاد (اگر محصول متغیّر بود)
         ↓
[9] Mapping ذخیره
         ↓
[10] ثبت در جدول "products_transferred"
         ↓
✅ انجام شد
```

## مثال عملی

### قبل (خرابی):
```
محصول: تیشرت خورشید
SKU: 115
Dategories: [5] (لباس زنانه)
Attributes: رنگ + سایز

نتیجه: ❌ خطای SKU + محصول ایجاد نشد
```

### بعد (درست):
```
محصول: تیشرت خورشید  
SKU: 115
Categories: [10] (دسته‌بندی بروز‌شده در سایت 2)
Attributes: رنگ + سایز (با تمام تکیه‌ها)

✅ محصول ایجاد شد
✅ 9 variation ایجاد شد
✅ در "محصولات منتقل‌شده" ظاهر شد
```

## راهنمای استفاده

### 1. اگر محصول منتقل نمی‌شود:
```
داشبورد → محصولات منتقل‌شده → لاگ‌ها
→ find `transfer_product_error`
→ خطا را بخوانید
```

### 2. اگر SKU وجود دارد:
```
✓ خودکار بروز‌رسانی می‌شود
✓ در لاگ "product_exists" نوشته می‌شود
✓ نیاز به عمل دستی نیست
```

### 3. اگر دسته‌بندی ایجاد نشد:
```
داشبورد → لاگ‌ها
→ filter by "sync_category"
→ error_message را بخوانید
```
