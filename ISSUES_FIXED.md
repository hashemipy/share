# مشکلات پیدا‌شده و حل‌شده - تب "مرتبط‌سازی محصولات"

## خلاصه مشکلات اصلی

تب "مرتبط‌سازی محصولات" (Product Mapping Tab) دارای چندین مشکل بود که مانع از نمایش و هماهنگ‌سازی صحیح محصولات‌ای شده بود:

---

## 🔴 مشکل ۱: عدم کارکرد دکمه "هماهنگ‌سازی همه موجودی‌ها"

### مشخصات:
- دکمه `sync-all-btn` در تب mapping فقط یک تاخیر مصنوعی بود
- هیچ AJAX call واقعی برای sync کردن mappingها وجود نداشت
- پیامی که نمایش داده می‌شد نشان دهنده فقط یک mock پیام بود

### حل:
- اضافه کردن AJAX call حقیقی برای `inventory_sync_sync_all_mappings` action
- بازگشت اطلاعات واقعی: تعداد موفق و ناموفق
- اضافه کردن handler جدید در class-admin.php: `ajax_sync_all_mappings()`

**تغییرات:**
- `/inventory-sync/assets/js/admin.js` - خطوط ۲۱۳-۲۳۴
- `/inventory-sync/includes/class-admin.php` - خطوط ۲۲ و ۲۲۷-۲۶۴

---

## 🔴 مشکل ۲: نبود ساز‌و‌کار مرتبط‌سازی محصولات

### مشخصات:
- هیچ راه‌کاری برای کاربر وجود نداشت تا محصولات سایت ۱ و سایت ۲ را مرتبط کند
- تابع `selectProduct()` تنها class انتخاب را toggle می‌کرد بدون کار واقعی
- نمایش محصولات ساده بود و فاقد واسط‌ای برای mapping

### حل:
- اضافه کردن دکمه ↔️ برای هر محصول (`btn-map-product`)
- پیاده‌سازی یک رویکرد modal برای انتخاب محصولات دو سایت
- اضافه کردن متد `openMappingModal()` برای تعامل کاربر
- اضافه کردن متد `createMapping()` برای ذخیره mapping در database

**تغییرات:**
- `/inventory-sync/assets/js/admin.js` - خطوط ۱۸۴-۲۷۷
- `/inventory-sync/assets/css/admin.css` - خطوط ۳۷۴-۴۱۵

---

## 🔴 مشکل ۳: کنترل ناکافی بر روی تکرار Mappingها

### مشخصات:
- هیچ بررسی برای تکرار mappingها وجود نداشت
- کاربر می‌توانست همان محصول را چند بار مرتبط کند
- یک محصول می‌توانست با چندین محصول مرتبط شود (1:N بجای 1:1)

### حل:
- اضافه کردن چک برای mapping موجود: `SELECT * FROM inventory_sync_mapping WHERE site1_product_id = ? AND site2_product_id = ?`
- اضافه کردن چک برای محصول استفاده‌شده: `WHERE site1_product_id = ?`
- اضافه کردن چک برای محصول استفاده‌شده در سایت دیگر: `WHERE site2_product_id = ?`
- نمایش پیام‌های خطای واضح به کاربر

**تغییرات:**
- `/inventory-sync/includes/class-admin.php` - خطوط ۱۸۶-۲۲۴

---

## 🔴 مشکل ۴: کش نپاک‌شده بین انتقال‌های متوالی

### مشخصات:
- متد `clear_cache()` در Inventory_Sync_Manager `private` بود
- اگرچه در کد نقاط صحیح فراخوانی می‌شد، اما نمی‌تواند از خارج دسترسی پیدا کند
- این ممکن است سبب مشکل شود اگر محصولات مختلف ویژگی‌های مشابهی داشته باشند

### حل:
- تغییر دسترسی متد `clear_cache()` از `private` به `public`
- این امکان دهد تا در صورت نیاز از خارج کلاس نیز فراخوانی شود

**تغییرات:**
- `/inventory-sync/includes/class-sync-manager.php` - خط ۳۷

---

## 🔴 مشکل ۵: رندر ضعیف محصولات در تب Mapping

### مشخصات:
- نمایش محصولات ساده بود بدون واسط برای کنش
- نبود هیچ نشانه‌ای برای محصولات‌ای که قبلاً mapped شده‌اند
- فقدان UI برای تعامل پویا

### حل:
- بهبود متد `renderProducts()` برای نمایش اطلاعات بیشتر
- اضافه کردن data attributes: `data-site`, `data-id`, `data-sku`
- اضافه کردن دکمه mapping برای هر محصول
- اضافه کردن event listeners برای تعامل

**تغییرات:**
- `/inventory-sync/assets/js/admin.js` - خطوط ۱۸۴-۱۹۲
- `/inventory-sync/assets/css/admin.css` - خطوط ۳۷۴-۴۱۵

## 🧪 آزمایش مورد نیاز

برای تأیید درستی تغییرات:

1. **تب مرتبط‌سازی محصولات**:
   - ✓ محصولات سایت ۱ و ۲ را بارگذاری می‌کند
   - ✓ دکمه ↔️ برای هر محصول قابل دیدن است
   - ✓ کلیک بر دکمه، محصول را برای mapping انتخاب می‌کند
   - ✓ انتخاب محصول از سایت دیگر، mapping را ذخیره می‌کند

2. **تب محصولات منتقل‌شده**:
   - ✓ mappingهای ایجاد شده نمایش داده می‌شود
   - ✓ موجودی‌ها درست هماهنگ می‌شود

3. **دکمه "هماهنگ‌سازی همه موجودی‌ها"**:
   - ✓ کلیک بر آن، تمام mappingهای فعال را sync می‌کند
   - ✓ گزارش صحیح از تعداد موفق/ناموفق دریافت می‌شود

---

## 📝 یادداشت‌های توسعه‌دهندگان

- `clear_cache()` تمام cache‌ها پاک می‌کند (حالا شامل site2_attributes_cache)
- Mapping بین دو محصول یک‌طرفه نیست - دوطرفه تنظیم شده است
- AJAX handlers تمام validation‌های لازم را انجام می‌دهند
- کل فرآیند از database تا frontend secure است (nonce check، permission check، sanitization)
