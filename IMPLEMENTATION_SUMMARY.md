# خلاصه پیاده‌سازی - انتقال خودکار دسته‌بندی و ویژگی‌ها

## مشکلات حل‌شده

### مشکل 1: دسته‌بندی موجود نیست در سایت 2
**قبل:** هنگام انتقال محصول، اگر دسته‌بندی در سایت 2 نبود، انتقال ناموفق می‌شد.
**بعد:** دسته‌بندی‌های محصول خودکار منتقل می‌شوند و اگر موجود نبودند، ایجاد می‌شوند.

### مشکل 2: ویژگی‌های محصول موجود نیستند
**قبل:** متغیّرهای محصول (رنگ، سایز) بدون ویژگی در سایت 2 ایجاد می‌شدند.
**بعد:** تمام ویژگی‌ها و تکیه‌های آن خودکار منتقل می‌شوند.

### مشکل 3: نمی‌دانیم کدام محصول منتقل شده است
**قبل:** هیچ علامتی برای محصولات منتقل‌شده نبود.
**بعد:** تب جدید "محصولات منتقل‌شده" با علامت ✓ برای هر محصول موفق.

### مشکل 4: ممکن است محصول دوباره انتقال پیدا کند
**قبل:** هیچ مکانیزمی برای جلوگیری از تکرار انتقال نبود.
**بعد:** محصولات منتقل‌شده در دیتابیس ثبت می‌شوند.

## ساختار تکنیکی

### 1. کلاس جدید: `Inventory_Sync_Category_Attribute_Sync`

**مسئولیت:**
- انتقال دسته‌بندی‌های محصول به صورت بازگشتی (والدین را هم منتقل کنید)
- انتقال ویژگی‌های محصول و تمام تکیه‌های آن‌ها

**متدهای اصلی:**

```php
// انتقال دسته‌بندی‌ها (بازگشتی - والدین را هم منتقل می‌کند)
sync_product_categories($product_categories): array

// انتقال ویژگی‌ها
sync_product_attributes($product_attributes): array

// جستجو برای دسته‌بندی موجود
find_existing_category($category_name, $parent_id): array|null

// جستجو برای ویژگی موجود
find_existing_attribute($attribute_name): array|null

// انتقال تکیه‌های ویژگی (رنگ، سایز و غیره)
sync_attribute_terms($site1_id, $site2_id, $attribute_name): void
```

**منطق کار:**

```
1. دریافت دسته‌بندی از سایت 1
   ↓
2. اگر دسته‌بندی والد دارد، ابتدا والد را منتقل کن (بازگشتی)
   ↓
3. جستجو برای دسته‌بندی موجود در سایت 2
   ├─ اگر موجود: استفاده از دسته‌بندی موجود
   └─ اگر نه: ایجاد دسته‌بندی جدید
   ↓
4. ثبت نقشه‌برداری (Mapping) در دیتابیس
```

### 2. جداول دیتابیسی جدید

#### `inventory_sync_category_mapping`
```
id                          - شناسه رکورد
site1_category_id           - شناسه دسته‌بندی در سایت 1
site2_category_id           - شناسه دسته‌بندی در سایت 2
site1_category_name         - نام دسته‌بندی سایت 1
site2_category_name         - نام دسته‌بندی سایت 2
site1_parent_id             - والد دسته‌بندی سایت 1
site2_parent_id             - والد دسته‌بندی سایت 2
sync_status                 - وضعیت (success/failed)
created_at / updated_at     - زمان ایجاد و ویرایش
```

#### `inventory_sync_attribute_mapping`
```
id                          - شناسه رکورد
site1_attribute_id          - شناسه ویژگی سایت 1
site2_attribute_id          - شناسه ویژگی سایت 2
site1_attribute_name        - نام ویژگی سایت 1
site2_attribute_name        - نام ویژگی سایت 2
attribute_type              - نوع ویژگی (pa / custom)
sync_status                 - وضعیت
created_at / updated_at     - زمان
```

#### `inventory_sync_products_transferred`
```
id                          - شناسه رکورد
site1_product_id            - شناسه محصول سایت 1
site2_product_id            - شناسه محصول سایت 2
product_name                - نام محصول
transferred_at              - زمان انتقال
transfer_status             - وضعیت (success/failed)
categories_synced           - وضعیت انتقال دسته‌بندی
attributes_synced           - وضعیت انتقال ویژگی
error_message               - پیام خطا (اگر موجود)
created_at                  - زمان ایجاد
```

### 3. بهبود متدهای API

فایل: `class-api.php`

**متدهای جدید برای دسته‌بندی:**
```php
get_categories($per_page, $page)           - دریافت لیست دسته‌بندی‌ها
get_category($category_id)                 - دریافت یک دسته‌بندی
create_category($category_data)            - ایجاد دسته‌بندی جدید
```

**متدهای جدید برای ویژگی:**
```php
get_attributes()                           - دریافت تمام ویژگی‌ها
get_attribute($attribute_id)               - دریافت یک ویژگی
create_attribute($attribute_data)          - ایجاد ویژگی جدید
get_attribute_terms($attribute_id, ...)    - دریافت تکیه‌های ویژگی
create_attribute_term($attr_id, $data)    - ایجاد تکیه جدید
```

### 4. بهبود Sync Manager

فایل: `class-sync-manager.php`

**تغییرات در `transfer_product($site1_product_id)`:**

```
1. دریافت محصول کامل از سایت 1
   ↓
2. ایجاد شیء Category_Attribute_Sync
   ↓
3. منتقل کردن دسته‌بندی‌ها
   ├─ همراه با والدین
   ├─ جستجو برای موجود
   └─ ایجاد اگر لازم بود
   ↓
4. منتقل کردن ویژگی‌ها
   ├─ همراه با تمام تکیه‌های آن
   ├─ جستجو برای موجود
   └─ ایجاد اگر لازم بود
   ↓
5. آپدیت Mapping‌های دسته‌بندی و ویژگی
   ↓
6. ایجاد محصول در سایت 2
   ├─ با دسته‌بندی‌های جدید
   ├─ و ویژگی‌های جدید
   └─ و تصاویر
   ↓
7. اگر محصول متغیّر است، منتقل کن:
   ├─ تمام Variations
   ├─ ویژگی‌های هر Variation
   └─ موجودی و قیمت
   ↓
8. ثبت محصول در جدول Transferred Products
   ↓
9. لاگ کردن موفقیت
```

## مثال عملی

### محصول: تیشرت خورشید

**در سایت 1:**
```
نام: تیشرت خورشید
نوع: متغیّر
دسته‌بندی: 
  - تیشرت تک (فرزند)
  - لباس زنانه (والد)
ویژگی‌ها:
  - رنگ: سبز (#1), سفید (#2), قرمز (#3)
  - سایز: 35 (#1), 40 (#2), 45 (#3)
Variations:
  - سبز-35 (Stock: 10)
  - سبز-40 (Stock: 15)
  - سفید-35 (Stock: 20)
  - ... و غیره
```

**فرایند انتقال:**
```
1. ابتدا دسته‌بندی والد "لباس زنانه" منتقل شود
2. سپس دسته‌بندی "تیشرت تک" منتقل شود (با والد صحیح)
3. ویژگی "رنگ" ایجاد شود + تکیه‌های آن
4. ویژگی "سایز" ایجاد شود + تکیه‌های آن
5. محصول والد ایجاد شود
6. تمام Variations ایجاد شوند
7. محصول در "محصولات منتقل‌شده" ظاهر شود
```

**نتیجه در سایت 2:**
```
✅ تیشرت خورشید
   - دسته‌بندی: تیشرت تک / لباس زنانه ✓
   - ویژگی‌ها: رنگ ✓, سایز ✓
   - Variations: 9 محصول متغیّر
   - Inventory: همراه با موجودی
   - تاریخ انتقال: 1403/3/25 14:30
```

## آزمایش و QA

### موارد آزمایشی پیشنهادی:

1. **انتقال محصول ساده:**
   - ✓ دسته‌بندی منتقل می‌شود
   - ✓ محصول ایجاد می‌شود
   - ✓ در جدول Transferred ثبت می‌شود

2. **انتقال محصول متغیّر:**
   - ✓ دسته‌بندی‌ها منتقل می‌شوند
   - ✓ ویژگی‌ها منتقل می‌شوند
   - ✓ تمام Variations منتقل می‌شوند
   - ✓ موجودی صحیح است

3. **دسته‌بندی موجود:**
   - ✓ اگر دسته‌بندی در سایت 2 موجود باشد، استفاده می‌شود
   - ✓ دسته جدید ایجاد نمی‌شود

4. **ویژگی موجود:**
   - ✓ اگر ویژگی در سایت 2 موجود باشد، استفاده می‌شود
   - ✓ ویژگی جدید ایجاد نمی‌شود

5. **جستجو در محصولات منتقل:**
   - ✓ نام محصول جستجو می‌شود
   - ✓ نتایج فیلتر می‌شوند

## نکات مهم

### سرعت:
- استفاده از Batch API برای انتقال متغیّرها
- بدون تأخیر برای انتقال دسته‌بندی‌ها

- Retry Logic برای API failures
- Error Logging کامل
- Atomic operations (یا هر چیز، یا هیچی)

- جستجوی محدود برای دسته‌بندی (نام و والد)
- Validation کامل برای ویژگی‌ها
