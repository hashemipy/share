# خلاصه پیاده‌سازی - انتقال خودکار ویژگی‌ها و دسته‌بندی‌ها

## 🎯 مسائل حل‌شده

### مسئله ۱: عدم انتقال دسته‌بندی‌های سلسله‌مراتبی
**وضعیت قبل:** ❌ خطای "دسته‌بندی وجود ندارد"
```
سایت 1: لباس کودک (مادر) → تیشرت تک (فرزند)
سایت 2: خطا! دسته‌بندی موجود نیست
```

**وضعیت بعد:** ✅ انتقال خودکار
```
سایت 1: لباس کودک → تیشرت تک
سایت 2: خودکار ایجاد می‌شود (والد → فرزند)
```

---

### مسئله ۲: عدم انتقال ویژگی‌ها و مقادیرشان
**وضعیت قبل:** ❌ خطای "ویژگی وجود ندارد"
```
سایت 1: 
  - رنگ: قرمز، سبز، آبی
  - سایز: ۳۵، ۴۰، ۴۵، ۵۰

سایت 2: خطا! ویژگی موجود نیست
```

**وضعیت بعد:** ✅ انتقال خودکار
```
سایت 1: 
  - رنگ: قرمز، سبز، آبی
  - سایز: ۳۵، ۴۰، ۴۵، ۵۰

سایت 2: خودکار ایجاد می‌شود (بصرف نظر از اینکه موجود باشد یا نه)
```

---

### مسئله ۳: انتقال محصولات تصادفی دوباره
**وضعیت قبل:** ⚠️ خطر انتقال دوباره
```
محصول منتقل شد ← (مدیر فراموش می‌کند) → دوباره منتقل می‌شود!
```

**وضعیت بعد:** ✅ علامت‌گذاری و جلوگیری
```
محصول منتقل شد ← علامت‌گذاری ✓ ← غیرفعال در انتقال
```

## 🔧 تغییرات فنی

### ۱. کلاس `Inventory_Sync_API` - متد‌های جدید

#### دسته‌بندی‌ها:
```php
// دریافت تمام دسته‌بندی‌ها
$api->get_categories($per_page, $page)

// جستجو براساس نام
$category = $api->get_category_by_name('لباس کودک')

// ایجاد دسته‌بندی جدید
$api->create_category($name, $parent_id)
```

#### ویژگی‌ها:
```php
// دریافت تمام ویژگی‌ها
$api->get_attributes($per_page, $page)

// جستجو براساس نام
$attribute = $api->get_attribute_by_name('رنگ')

// ایجاد ویژگی جدید
$api->create_attribute($name)

// دریافت مقادیر ویژگی (Terms)
$terms = $api->get_attribute_terms($attribute_id)

// جستجو برای مقدار خاص
$term = $api->get_attribute_term_by_name($attribute_id, 'قرمز')

// ایجاد مقدار جدید
$api->create_attribute_term($attribute_id, 'آبی')
```

#### متا‌داده:
```php
// به‌روز‌رسانی متا‌داده محصول (برای علامت‌گذاری)
$api->update_product_meta($product_id, '_inventory_sync_transferred', $data)
```

### ۲. کلاس `Inventory_Sync_Sync_Manager` - بهینه‌سازی

#### جریان `transfer_product()` جدید:
```php
transfer_product($site1_product_id)
    ├─ ۱. sync_categories()        // انتقال دسته‌بندی‌ها
    ├─ ۲. sync_attributes()         // انتقال ویژگی‌ها
    ├─ ۳. create_product()          // ایجاد محصول
    ├─ ۴. transfer_variations()     // انتقال متغیّرها (اگر متغیّر بود)
    ├─ ۵. save_mapping()            // ذخیره نقشه‌برداری
    └─ ۶. update_product_meta()     // علامت‌گذاری
```

#### متد `sync_categories()` - نیاز دارد:
```php
private function sync_categories($product1)
    └─ Category_Helper->sync_category_with_hierarchy($cat_id)
        ├─ دریافت سلسله‌مراتب کامل (والد ← فرزند)
        ├─ ایجاد والدین (اگر نبودند)
        └─ ایجاد فرزند (اگر نبود)
```

#### متد `sync_attributes()` - نیاز دارد:
```php
private function sync_attributes($product1)
    ├─ برای هر ویژگی:
    │  ├─ ایجاد ویژگی (اگر نبود)
    │  └─ ایجاد مقادیر ویژگی (اگر نبودند)
    └─ اگر یک مقدار مشکل داشت، ادامه می‌دهد
```

### ۳. کلاس `Inventory_Sync_Category_Helper` (جدید)

**مسئولیت:** مدیریت دسته‌بندی‌های سلسله‌مراتبی

```php
$helper = new Inventory_Sync_Category_Helper($site1_api, $site2_api)

// دریافت سلسله‌مراتب
$hierarchy = $helper->get_category_hierarchy($api, $category_id)
// نتیجه: [والد، فرزند، نوه، ...]

// انتقال با سلسله‌مراتب
$result = $helper->sync_category_with_hierarchy($cat_id)
// چک می‌کند هر دسته‌بندی موجود است یا نه
// دسته‌بندی‌های والد را ابتدا ایجاد می‌کند
// سپس فرزندان را ایجاد می‌کند
```

## 📋 شماره‌ای از درخواست‌های API

### هنگام انتقال ۱ محصول متغیّر:

```
درخواست ۱: GET /products/{id}
    └─ دریافت محصول از سایت ۱

درخواست ۲: GET /products/categories (صفحه ۱)
    └─ دریافت دسته‌بندی‌های والد/فرزند

درخواست ۳: GET /categories (در سایت ۲)
    └─ چک دسته‌بندی‌ها در سایت ۲

درخواست ۴: POST /categories
    └─ ایجاد دسته‌بندی مادر (اگر نبود)

درخواست ۵: POST /categories
    └─ ایجاد دسته‌بندی فرزند (اگر نبود)

درخواست ۶: GET /attributes
    └─ دریافت ویژگی‌های سایت ۱

درخواست ۷: GET /attributes (سایت ۲)
    └─ چک ویژگی‌ها در سایت ۲

درخواست ۸: POST /attributes
    └─ ایجاد ویژگی "رنگ" (اگر نبود)

درخواست ۹-۱۱: POST /attributes/{id}/terms
    └─ ایجاد مقادیر: قرمز، سبز، آبی

درخواست ۱۲: POST /attributes
    └─ ایجاد ویژگی "سایز" (اگر نبود)

درخواست ۱۳-۱۶: POST /attributes/{id}/terms
    └─ ایجاد مقادیر: ۳۵، ۴۰، ۴۵، ۵۰

درخواست ۱۷: POST /products
    └─ ایجاد محصول متغیّر

درخواست ۱۸: GET /products/{id}/variations
    └─ دریافت متغیّرها از سایت ۱

درخواست ۱۹: POST /products/{id}/variations/batch
    └─ ایجاد گروهی متغیّرها

درخواست ۲۰: PUT /products/{id}
    └─ علامت‌گذاری محصول اصلی
```

**کل: حدود ۲۰ درخواست برای ۱ محصول متغیّر** 🔄

## 🧪 آزمایش

### تست ۱: محصول ساده
```
1. انتخاب: محصول ساده (مثل کتاب)
2. دسته‌بندی: کتب > علمی
3. انتقال کن
4. بررسی:
   ✓ محصول در سایت ۲ ایجاد شد
   ✓ دسته‌بندی ها ایجاد شدند
   ✓ محصول علامت‌گذاری شد
```

### تست ۲: محصول متغیّر
```
1. انتخاب: محصول متغیّر (تیشرت با رنگ و سایز)
2. دسته‌بندی: لباس > تیشرت
3. ویژگی‌ها: 
   - رنگ: قرمز، سبز، آبی
   - سایز: ۳۵، ۴۰، ۴۵، ۵۰
4. انتقال کن
5. بررسی:
   ✓ دسته‌بندی‌های والد و فرزند ایجاد شدند
   ✓ ویژگی‌های رنگ و سایز ایجاد شدند
   ✓ تمام مقادیر ایجاد شدند
   ✓ تمام متغیّرها ایجاد شدند
   ✓ محصول علامت‌گذاری شد
```

### تست ۳: دوباره انتقال (جلوگیری)
```
1. محصول منتقل‌شده را انتخاب کن
2. چک کن: آیا غیرفعال است؟
3. امتحان کن: کلیک روی درخواست انتقال
4. بررسی:
   ✓ محصول نمی‌تواند دوباره انتخاب شود
   ✓ نشانگر "✓ منتقل شده" نمایش داده می‌شود
```

## 📝 نکات مهم

### هنگام استفاده:
1. **تنظیمات API را اول تست کنید** ⚠️
2. **با محصولات کم شروع کنید** (۱-۵ محصول)
3. **لاگ‌ها را بررسی کنید** اگر مشکلی بود

### محدودیت‌های شناخته‌شده:
- ⚠️ اگر سلسله‌مراتب بسیار عمیق است، ممکن است بطی شود
- ⚠️ اگر سایت API کند است، timeout ممکن است رخ دهد
- ⚠️ تصاویر محصول جداگانه منتقل می‌شوند (نه به‌صورت دستی)

### بهبود‌های آینده:
- 💡 Caching برای دسته‌بندی‌ها
- 💡 Batch API برای ویژگی‌های زیادتر
- 💡 تاریخچه تغییرات تفصیلی
